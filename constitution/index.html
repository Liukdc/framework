态控架构 · §七 约束体系

本文件是细则文件——对应总纲概要的 §七。版本 v5.5。

七、约束体系​

7.1 三层宪法体系总览​

按约束对象分层，每一层的约束对象唯一。

层级名称约束对象是否可修改说明
第0层根宪法所有环节模型（ANALYZING + IN_SESSION）设计者写死，不可运行时修改不可逾越的系统红线
第1层环节宪法特定环节模型设计者静态编写（默认）；开放型环节可运行时动态生成环节内模型的行为规则
第2层领域规则特定主题类型用户确认后生效，可随时修改或废弃L0-L3 四级权限梯度
—架构机制调度器/DET设计者定义元指令集、降级链、仲裁权、节点转换守卫——约束调度器和DET行为，非模型约束

7.2 根宪法（4 条，约束所有环节模型）​

定位：所有环节模型（ANALYZING + IN_SESSION）都必须遵守的底线条款。由设计者写死，不可运行时修改，不可被环节宪法覆盖。

编号条款内容可被环节宪法覆盖
1环节隔离只能做本环节任务，非本环节任务返回 off-task（偏离）。本环节任务由环节宪法第一条定义。ANALYZING（分析态）只能做意图识别。IN_SESSION（会话中）只能做本环节任务。❌ 不接受覆盖
2输出格式模型实例返回调度器的结果必须是符合 turnType schema 的结构化 JSON（含 askingField 双模式语义 + changeLevel 预判字段 + changeLevelReason 理由字段，两种 taskType 模式均适用）❌ 不接受覆盖
3收敛义务 + 坚持真实判断所有 LLM 节点必须在当前环节内收敛，输出结构化 JSON。不伪造数据、不假装理解、不迎合用户、不无中生有。确认无误就坚持，确实错了就纠正。❌ 不接受覆盖
4不编造功能所有 LLM 节点不得生成用户未交代、系统不具备的功能或信息。这是安全红线的真核心——不是场景特定的"禁止恶意内容"，而是全局硬约束"不许无中生有"。❌ 不接受覆盖

ANALYZING（分析态）的环节隔离特化：ANALYZING 用强制选择+API logprobs（非 LLM 自由生成 confidence）。如果 logprobs probability 不足以确认任何已知 intent，返回 intent=other（低 probability，调度器按 probability 路由）。ANALYZING 只产出意图识别相关信号（intent / choice / logprobs / inputNature / extracted / topicId），不产出 turnType——turnType 是 IN_SESSION 的输出格式。 turnType=off-task 是 IN_SESSION 内模型检测到偏离时的输出，不是 ANALYZING 的输出。ANALYZING 不得采集字段、不得追问、不得给出最终回复。

7.3 架构机制（约束调度器/DET，非模型约束）​

以下四条属于架构规范，约束调度器和 DET 行为：

编号机制约束对象内容
M1元指令集（四类口令 + S3 + roomStateIndex）调度器（第0层）+ 内存管理模块 + context-manager四类通用口令：wake / switch / exit / cancel。口令层 EXACT_MATCH→直切（0 次 LLM）。wake/switch/exit/cancel 触发 S3 物理 KV Cache 释放 + roomStateIndex 注入。cancel = 中断当前推理请求但不归档房间（区别于 exit 的"归档回 IDLE"），用于"用户想重来这一轮"的场景——释放当前轮 KV Cache，房间状态不变，重新等待用户输入。wake=创建新推理会话 + 从 contractStore 恢复 checkpoint + buildRoomIndex() 注入 ANALYZING；switch=释放当前房间 KV Cache + 归档 + 创建新会话 + 恢复目标房间 checkpoint + buildRoomIndex() 注入；exit=释放 KV Cache + 归档回 IDLE。roomStateIndex 提供全窗口房间摘要（各房间待完成/已完成数），使 ANALYZING 可针对性建议。设计者定义，DET+内存管理+context-manager 执行。完整 S3 行为详见 §9.4
M2降级链四项检查调度器/DETL1 结构校验（DET）+ DET 值域复验（DET，field_based值域/topic_based格式）+ logprobs 检查（DET）+ 硬编码兜底（DET）。均不可缩减
M3仲裁权 DET 优先调度器调度器 DET 校验结果优先于模型内部校验结果。两者冲突时以调度器 DET 为准
M4节点转换守卫调度器定义环节间转换的触发条件和守卫规则。IN_SESSION 期间用户输入偏离时，调度器 DET 关键词扫描可辅助触发偏离标记，但最终判断由模型给出

7.4 环节宪法（约束特定环节模型）​

  1. 第一条强制为首要内容：环节宪法第一条必须定义"本环节任务是什么"并强调"非本环节任务返回 off-task"。
  2. 追问模板下放到 @section ask-rules：追问模板（追问类型/主题/格式三维度）由各环节宪法的 @section ask-rules 自行定义。
  3. 环节宪法结构模板（18 标签 @section）： 
    <!-- @constitution <session_name> -->
    <!-- @section taskType -->
    <!-- @section role -->（第一条：本环节任务定义 + off-task 声明）
    <!-- @section convergence -->
    <!-- @section field-rules | topic-boundary -->
    <!-- @section parsing -->
    <!-- @section validation -->
    <!-- @section ask-rules -->（含追问模板）
    <!-- @section completion -->
    <!-- @section modification -->
    <!-- @section off-task-detection -->
    <!-- @section giveup -->
    <!-- @section output-schema -->
    <!-- @section tunable -->
    <!-- @section domain-rule-priority -->
    <!-- @section scale -->（并联实例化配置，可选）
    <!-- @importance -->
    <!-- @outputs -->

7.5 宪法仲裁流程​

环节宪法条款与根宪法冲突
    ↓
反对称仲裁（根宪法无条件优先）
    ↓
冲突条款以根宪法为准，环节宪法对应条款标记为"不可用"

因公共规则已取消，仲裁流程为三层（根/局部/领域）。
