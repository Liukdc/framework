## §0：P0 认知加载

### 一句话场景定义
极简碎片信息记录工具。用户说出碎片信息，系统记录并按用户设定时间提醒整理，长期不处理后自动清理。

### 八根支柱检查

| 支柱 | 内容 | P0 判断 |
|------|------|---------|
| 1 认知局部性 | 每条记录独立，不推断关联 | ✅ 核心约束 |
| 2 意图收敛 | 5 个 intent（record/search/organize/setting/other）闭环 | ✅ 可穷举 |
| 3 稳态预设 | IDLE 为唯一稳态，所有流程回归 IDLE | ✅ 已定义 |
| 4 分级熔断 | 超时→默认值→IDLE；放弃→IDLE | ✅ 三级 |
| 5 信息压缩 | 原始输入唯一数据源，不概括 | ✅ 硬约束 |
| 6 状态隔离 | 11 个 state，每个只做一件事 | ✅ 已设计 |
| 7 确定性优先 | 唤醒词/退出词 DET 短路；Key 格式 DET 硬解析 | ✅ 零 LLM |
| 8 边界先行 | 不分类/不打扰/不丢失 三条铁律 | ✅ 写入宪法 |

### P0 结论
全部 field_based，5 intent 可直接穷举。**N2 跳过**（field_based 场景，字段即边界，无语义模糊空间）。

## §1：N1 场景定义与边界清单

### 1.1 边界清单

| 维度 | 做什么 | 不做什么（含处理策略） | taskType |
|------|--------|----------------------|----------|
| 核心功能 | 记录碎片信息 | 不分类/不贴标签/不分析内容 — 硬拒 | field_based |
| 核心功能 | 按 Key 精确检索 | 不语义搜索/不模糊匹配 — 硬拒 | field_based |
| 核心功能 | 定时提醒整理 | 不替用户分类/不自动生成摘要 — 硬拒 | field_based |
| 核心功能 | 自动清理废弃记录 | 不提前删除未到期记录（除非用户手动） — 硬拒 | field_based |
| 交互方式 | 语音唤醒 + 文字输入 | 不支持图片识别/手势操作 — 硬拒 | — |
| 数据存储 | 本地存储，支持多模态附件 | 不上传云端分析/不分享给第三方 — 硬拒 | — |
| 用户角色 | 个人用户，每天几条到十几条 | 用户无需注册/登录/授权 — 无需处理 | — |

### 1.2 意图清单（doList）

| intent | description | taskType | topicEvolutionEnabled | interactionMode | dataHandling |
|--------|-------------|----------|----------------------|-----------------|--------------|
| record | 录入碎片信息：Key→Content→附件→整理时间 | field_based | false | 语音/文字 | 本地文件存储 |
| search | 按 Key 精确检索已有记录 | field_based | false | 语音/文字 | 本地数据库查询 |
| organize | 逐条整理未命名/未确认的记录 | field_based | false | 语音/文字 | 本地数据库更新 |
| setting | 修改提醒周期/默认整理天数等配置 | field_based | false | 文字 | 本地配置写入 |
| other | 闲聊/感谢/听不懂 — 回流引导 | field_based | false | 语音/文字 | 无 |

### 1.3 设计铁律（三条）

1. **不分类**：系统不替用户做任何分类、标签或智能分析
2. **不打扰**：提醒仅一次，用户拒绝或超时即取消
3. **不丢失**：废弃记录先进回收区，保留 30 天

### 1.4 松弛节点声明
默认不启用。

## §4：N4 路由表

### 4.1 状态转移图（11 nodes）

```
IDLE ──[唤醒词"杂碎本"]──→ LISTENING

LISTENING ──[用户输入]──→ ANALYZING

ANALYZING ──[intent=record]──→ RECORDING
ANALYZING ──[intent=search]──→ SEARCHING
ANALYZING ──[intent=organize]──→ ORGANIZING
ANALYZING ──[intent=setting]──→ SETTING
ANALYZING ──[intent=other]──→ IDLE (输出引导语)

RECORDING ──[字段齐全]──→ VALIDATING
RECORDING ──[超时/放弃]──→ IDLE
RECORDING ──[needConfirm=true]──→ CONFIRMING

VALIDATING ──[校验通过]──→ EXECUTING
VALIDATING ──[校验失败]──→ RECORDING (回退追问)

EXECUTING ──[保存完成]──→ IDLE

SETTING ──[配置完成/取消]──→ IDLE

CONFIRMING ──[用户确认]──→ EXECUTING
CONFIRMING ──[用户取消]──→ IDLE

SEARCHING ──[匹配/取消]──→ IDLE

ORGANIZING ──[队列走完/退出]──→ IDLE

CLOSING ──[完成]──→ IDLE
```

### 4.2 模型选择

| 层 | 绑定 | 模型 |
|----|------|------|
| 第一层 | ANALYZING | 通用轻量模型 |
| 第一层 | RECORDING | 通用模型 |
| 第二层 | intent=record | 通用模型 |
| 第二层 | intent=search/organize/setting/other | 通用模型（DET，无 LLM） |

## §5：N5 调度器核心逻辑

### 5.1 三层处理链

```
用户输入
 │
 ├─ 第0层：全局硬匹配（DET，0 LLM）
 │   唤醒词"杂碎本" → LISTENING
 │   退出词"算了/不记了/退出" → IDLE
 │   命中 → 直接转 state
 │
 ├─ 第1层：录入态（LLM 逐字段采集）
 │   RECORDING → LLM 按字段顺序逐字段询问采集
 │   字段齐全 → VALIDATING
 │   超时/放弃 → IDLE
 │
 └─ 第2层：LISTENING 态 → 调 LLM
      拼接意图识别宪法 → LLM 返回 {intent, confidence, extracted}
      → 状态机根据 intent 分发
```

### 5.2 上下文管理器接口

| 接口 | 调用时机 | 说明 |
|------|---------|------|
| buildPromptContext() | ANALYZING 前 | 注入意图识别宪法 + collectedFields |
| updateGraph() | 每轮 LLM 返回后 | 更新 turnHistory |
| archiveGraph() | 偏离/放弃/IDLE | 归档当前上下文 |

### 5.3 五项降级链

| 检查 | 执行体 | 时机 | 失败动作 |
|------|--------|------|---------|
| L1 结构校验 | DET | LLM 返回后 | JSON.parse 失败→重试 1 次→硬编码回复 |
| DET 值域复验 | DET | L1 通过后 | intent 不在五值内→VALIDATING→RECORDING |
| confidence 检查 | DET | 值域通过后 | < 40 → VALIDATING→RECORDING（冷启动期软拦截） |
| 跨任务延伸检测 | DET | confidence 通过后 | 字段白名单（纯 field_based，needSemanticExtensionCheck=false） |
| 硬编码兜底 | DET | 熔断时 | 硬编码回复→IDLE |

## §7：N7 根宪法

```
第1条（安全红线）：不上传用户数据，不分享给第三方，不用于训练。
第2条（元指令）：用户原始输入是唯一数据源。不修改、不概括、不解释。
第3条（收敛义务）：每次只做一件事。一个 intent = 一次 LLM 调用。
第4条（输出格式）：turnType 六值 + askingField + changeLevel + changeLevelReason + message + collectedFields。
第5条（跨任务延伸检测）：field_based 场景使用字段白名单，needSemanticExtensionCheck=false。
第6条（状态间数据传递）：严格遵守 N6 传递协议，不跨状态携带未声明数据。
第7条（三类保护机制）：原始输入保护（不修改）+ 草稿保护（不清理未到期记录）+ 数据保护（不上传云端）。
第8条（模糊目标处理）：用户说"记一下"但未提供 Key→自动生成临时 Key。用户说"找东西"但未提供精确 Key→反问引导。
```

---

## §8：N8 公共规则（3 条）

```
规则1（输出格式）：输出必须是纯 JSON。turnType 六值齐全。
规则2（角色真实）：你是杂碎本，极简碎片记录助手。30 字以内。禁止延伸/建议/"您还可以"/"如果需要"。
规则3（跨任务延伸检测）：field_based 场景，needSemanticExtensionCheck=false。
```

## §9：N9 环节宪法

### 9.1 意图识别宪法（LISTENING→ANALYZING 注入）

```json
{
  "intent": "intent-recognition",
  "taskType": "field_based",
  "needSemanticExtensionCheck": false,
  "topicEvolutionEnabled": false,
  "@importance": "high",
  "@importance:deliverable": {"extracted": "critical"},
  "outputSchema": {
    "turnType": "ask | reply | complete | off-task | giveup | validation_failed",
    "askingField": "intent",
    "changeLevel": "major | minor | invalid",
    "changeLevelReason": "用于修改日志记录"
  },
  "fieldRules": {
    "intent": {"method": "enum", "values": ["record","search","organize","setting","other"], "error_tag": "intent_invalid"},
    "confidence": {"method": "range", "min": 0, "max": 100, "error_tag": "confidence_out_of_range"},
    "extracted.key": {"method": "nullable", "error_tag": "key_extraction_failed"},
    "extracted.content": {"method": "nullable", "error_tag": "content_extraction_failed"}
  },
  "completionCondition": "intent ∈ 五值 + confidence ∈ [0,100] + 结构完整"
}
```
