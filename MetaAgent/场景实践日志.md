**开始日期**：2026-07-11 | **最后更新**：2026-07-12

---

## 一、项目概述

### 1.1 做什么

造一个"智能体设计助手"——用户说"我想做一个记账智能体"，元智能体按 16 步流程（P0→N1→N3→...→N15）引导用户完成从边界划定到交付的全流程设计。

### 1.2 核心特征

- 16 个 intent 全部 **field_based**（每步产出为预定义字段集，DET 可验）
- 全部 **topicEvolutionEnabled=false**（不启用主题演化）
- **异步规则提炼独立启用**——按 stepName 映射，field_based 场景同样适用
- **N2 跳过**——纯 field_based 场景，字段即边界，无语义模糊空间

### 1.3 方法论应用

基于态控架构 v4.2 + 流程指南 v7.1 + 节点说明 v5.1，逐节点（N1→N15）产出节点说明 + 环节宪法，每个节点对照基准版进行适配。

## 三、关键决策记录

### 决策 1：N2 跳过
- **时间**：2026-07-12
- **内容**：纯 field_based 场景跳过 N2 边界紧张度测试
- **理由**：字段本身就是边界——字段映射成功就是类一，映射失败就是类二，没有灰色地带。N2 的每一项测试（逐条判类/护栏响应质量/边界覆盖完整性/topicEvolution 跨会话/双 confidence）对 field_based 都多余
- **影响**：N1→N3 直连。N1 语料验证移至 N3 环节内用户逐条确认。N16 审计链需标记 n2Skipped=true

### 决策 2：异步规则提炼——field_based 独立启用
- **时间**：2026-07-12
- **内容**：规则提炼不依赖 topicEvolution，直接按 stepName 映射。field_based 场景同样适用
- **理由**：元智能体 16 步虽为 field_based，但每步内模型与用户交互自由度大——同一个 N5 环节，不同用户的设计需求差异显著。规则提炼可以从多次执行同一环节的经验中提取最佳实践
- **影响**：N10 独立声明 4 个规则提炼参数（domain_rule_validation_threshold=0.85 等）。N3/N4/N5/N6 需在上下文注入时支持 stepName→规则映射

### 决策 3：WAITING_CONFIRM——每步保存都需确认
- **时间**：2026-07-12
- **内容**：每步产出保存前过 WAITING_CONFIRM，用户确认后方可写入。修改已有产出时额外提示"已有产出将被覆盖"并记录修改日志
- **理由**：每一步产出均为设计文档/代码文件，保存即写盘。修改日志作为异步规则提炼的素材（用户为什么改、改成什么样）
- **影响**：最优路径从 IN_SESSION→EXECUTING 变为 IN_SESSION→WAITING_CONFIRM→EXECUTING。CLOSING 双出口均保留

### 决策 4：N3 状态集——8 框架 + 16 子类型
- **时间**：2026-07-12
- **内容**：保留 8 个框架状态（IDLE/LISTENING/ANALYZING/IN_SESSION/EXECUTING/CLOSING/CLARIFYING/WAITING_CONFIRM），省略 6 个可选状态
- **省略状态及理由**：SLACK_NODE（N1 enabled=false）/ PARAM_PARSING（合并入 ANALYZING）/ VALIDATING（合并入 IN_SESSION）/ PREVIEWING（无需可视化 diff）/ CONSTITUTION_GEN（静态宪法）/ CONFIRM_CONSTITUTION（不适用）

## 四、与基准版的关键差异

### N3

| 基准（v5.5） | 元智能体（v2.0） | 原因 |
|-------------|----------------|------|
| topic_based | field_based | 16 步均为预定义字段集 |
| 消费 N2 测试报告 | n2Skipped=true | 字段即边界 |
| topicEvolutionConfig 必填 | 无 | 全部 false |
| 少数派拆分逻辑 | 无 | 全部同一 taskType |
| changeLevel/changeLevelReason | 无 | field_based 不适用 |

### N4

| 基准（v5.6） | 元智能体（v2.0） | 原因 |
|-------------|----------------|------|
| topic_based | field_based | 同上 |
| topicEvolution 路由配置 | 无 | 全部 false |
| 软着陆路由 | 无 | N1 softLandingTargets 为空 |
| 松弛节点 chitchat_path | 无 | N1 enabled=false |
| expectsChangeLevel 标注 | 无 | field_based 不适用 |
| 最优路径直达 EXECUTING | 过 WAITING_CONFIRM | 每步保存需确认 |
