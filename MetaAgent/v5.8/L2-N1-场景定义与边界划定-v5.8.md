# L2-N1 场景定义与边界划定（v5.8）

**项目**：元智能体（MetaAgent）
**基线**：态控架构 v5.8 / N1节点说明 v7.0
**日期**：2026-07-20

## 一、HumanReadable

### 1. 场景定义

"这是一个智能体开发引导助手。用户说出唤醒词，系统通过对话引导用户完成从P0到N15的完整态控设计流程，输出L2流程文档和L3生成包。"

### 2. 意图清单

| intent | 描述 | taskType | topicEvolutionEnabled |
|--------|------|----------|----------------------|
| P0 | 认知加载——确保用户理解态控核心概念 | topic_based | true |
| N1 | 场景定义与边界划定 | topic_based | true |
| N2 | 边界紧张度测试（双角色串行） | topic_based | true |
| N3 | 状态枚举+执行体分派+taskType固化 | topic_based | true |
| N4 | 状态转移图+路由表 | topic_based | true |
| N5 | 调度器核心逻辑+上下文管理器三接口 | topic_based | true |
| N6 | 数据传递协议+守卫规则 | topic_based | true |
| N7 | 根宪法4条+架构机制M1-M4 | topic_based | true |
| N8 | 架构机制核查 | topic_based | true |
| N9 | 环节宪法（@section tools/@section tool_catalog） | topic_based | true |
| N10 | tunable参数声明 | topic_based | true |
| N11 | 契约对齐（27项检查） | field_based | false |
| N12 | L2→L3拆包 | field_based | false |
| N13 | 骨架生成（代码专项模型） | topic_based | true |
| N14 | 审骨架+20+1case+9+2机制 | topic_based | true |
| N15 | 调参交付 | topic_based | true |
| other | 无法归类 | topic_based | true |

**taskType 说明**：MetaAgent 本质是 topic_based（引导对话），但 N11/N12 产出为结构化字段集，按 field_based 处理。

### 3. 边界清单

| 维度 | 做什么 | 不做什么 |
|------|--------|---------|
| 核心功能 | P0~N15共16步设计流程引导 | 不在16个节点之内→引导性硬拒回当前环节 |
| 交互方式 | 纯文本对话引导 | 不支持图形化拖拽（硬拒） |
| 对话范围 | 仅限智能体设计相关讨论 | 不闲聊（硬拒） |
| 数据处理 | 会话内暂存设计中间态+sessionCheckpoint跨会话恢复 | 不上传云端 |
| 输出物 | L2文档 + L3生成包 | 不直接生成可运行代码（N13是代码模型的事） |

### 4. 模型选择

- 第一层：ANALYZING→DeepSeek轻量 / IN_SESSION→DeepSeek通用
- 第二层：N13→代码专项模型

### 5. 护栏策略

- 引导性硬拒："当前在{{stepName}}环节，请先完成这一步。"
- 松弛节点：不启用

### 6. 语料计划

- targetCount: 20-30, 模式B（模型生成）

### 7. tunable参数

通用16+topic_based4+v5.7/v5.8新增5=25个适用

## 三、v2.0 → v5.8 关键变化

| 维度 | v2.0(v4.7) | v5.8 |
|------|-----------|------|
| taskType | 错误标注为全field_based | **14个topic_based + 2个field_based** |
| N2 | 错误跳过 | **必须执行**(有topic_based intent) |
| topicEvolution | 未启用 | **topic_based intent自动启用** |
| 降级链 | 五项(含跨任务延伸检测) | **四项** |
| confidence | 使用 | **logprobs替代** |
| 松弛节点 | 不启用 | 不启用(与v2.0一致) |
| 第二分类维度 | 无 | **S/T/U路由** |
