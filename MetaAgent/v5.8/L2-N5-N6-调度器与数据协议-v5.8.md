## N5：调度器核心逻辑

### 五层过滤（MetaAgent特有）

| 层 | MetaAgent特有 | 说明 |
|----|-------------|------|
| 第0层 | M1口令（wake/exit/cancel/switch） | 同富贵小安 |
| 第0.5层 | DET关键词扫描 | topic_based不需要关键词扫描（无跨意图切换概念——设计讨论本身就是开放的） |
| 第1层 | ANALYZING | 强制选择(A~R)+logprobs。17个intent（P0+N1-N15+other） |
| 第2层 | IN_SESSION | **topic_based**：三层注入（领域规则+关联摘要+topicEvolution分层包）+ 模型自主讨论 |
| 第3层 | DET四项检查 | topic_based用输出格式校验，field_based(N11/N12)用值域校验 |

### MetaAgent特有逻辑

| 机制 | 实现 |
|------|------|
| **N2双角色串行** | N5检测到intent=N2 → 不走普通IN_SESSION → 调度器直调角色一（只注场景定义）→ DET污染检测 → 直调角色二（注角色一输出+N1边界+意图） |
| **WAITING_CONFIRM** | topic_based+topic匹配成功 → "检测到已有主题[XX]，继续还是新建？" |
| **topicEvolution** | 每个topic_based环节complete → 追加event（changeLevel=预判值）→ stateChange摘要N10后台异步生成 |
| **N13代码模型** | intent=N13 → 第二层模型绑定 → 代码专项模型 |
| **sessionCheckpoint** | N11/N12 → field_based → sessionCheckpoint记录进度 |

## N6：数据传递协议

### 关键转移路径

| 路径 | topic_based携带 | field_based携带 |
|------|---------------|----------------|
| ANALYZING→IN_SESSION | intent+topicId+taskType+logprobs+extracted | N11/N12同左 |
| ANALYZING→WAITING_CONFIRM | topicId+candidateName+keywords+matchResult | — |
| IN_SESSION→LISTENING | 追加topicEvolution event(changeLevel) | N11/N12追加sessionCheckpoint |
| IN_SESSION→ANALYZING(off-task) | offTaskInput+暂存collectedFields+追加event(active) | 同左 |
| IN_SESSION→LISTENING(giveup) | 追加event(abandoned)+推入提炼队列 | 清空activeSession |

### topicEvolution存储

- topicId: `{userId}_{intent}_{topicSlug}_{date}`
- changeLevel: major/minor/invalid（模型预判，格式错误默认minor）
- 分层留存: major永久/minor压缩(5条→1)/invalid归档
- 注入包: majorEvents+compressedMinor+recentMinor+currentSnapshot

### contractStore表

12张标准表（同富贵小安）+ topicEvolution/topicEvolutionEvent/topicEvolutionArchive/topicIndex（topic_based专属）
