## 四、产出物

N9 产出：16 份宪法文件——15 份 IN_SESSION 环节宪法 + 1 份 ANALYZING 意图识别宪法（analyzing-session.json）。N2 跳过（纯 field_based），无宪法文件。

### 4.1 宪法模板（field_based）

全部 16 份宪法使用统一模板——14 个必填标签 + 1 个可选标签。

| # | @section 标签 | 必填 | 说明 |
|---|-------------|------|------|
| 1 | @section constitution | ✅ | taskType=field_based 声明 + 收敛方向 |
| 2 | @section role | ✅ | 执行体角色定义 + 上游消费顺序 |
| 3 | @section convergence | ✅ | 产出物清单 |
| 4 | @section ask-rules | ✅ | 引导式追问规则（分阶段，每阶段含 askingField） |
| 5 | @section output-schema | ✅ | turnType 六值 + changeLevel/changeLevelReason（用于修改日志） |
| 6 | @section completion | ✅ | 语义检查 + 静默自检，全部通过后 complete |
| 7 | @section validation | ✅ | 输出校验规则（required_fields + rules 数组） |
| 8 | @section modification | ✅ | 修改规则——用户改XX→回到对应位置 |
| 9 | @section off-task-detection | ✅ | 偏离检测——用户说不在本环节的话怎么处理 |
| 10 | @section giveup | ✅ | 放弃规则——用户说"算了""不做了" |
| 11 | @section field-rules | ✅ | 字段级校验规则表（哪些字段怎么验） |
| 12 | @section parsing | ✅ | 字段解析规则 |
| 13 | @section domain-rule-priority | ✅ | 异步规则提炼注入声明——同一 stepName 再次执行时注入已生效领域规则 |
| 14 | changeLevel/changeLevelReason | ✅ | output-schema 内声明——major=核心设计变化 / minor=渐进调整 / invalid=无效变更。changeLevelReason 1-100字具体理由。空值/超100字→DET拦截+默认"未提供"+记录警告，触发重试或进入 CLARIFYING。**全部宪法必填——用于修改日志记录，供 N10 异步规则提炼消费** |
| — | @section tunable | 可选 | 环节特有 tunable 参数声明（规则提炼参数引用） |

**省略标签**：@section topic-boundary（field_based 不需要话题边界，偏离检测由 off-task-detection 管）。

### 4.2 analyzing-session.json（ANALYZING 节点宪法）

ANALYZING 节点负责 16 步意图识别，其宪法定义：

| 标签 | 内容 |
|------|------|
| @section constitution | field_based——16 步意图匹配为确定性字段映射 |
| @section role | 读取 N1 意图清单（16 intent），逐条匹配用户输入 |
| @section output-schema | 输出 intent + taskType=field_based + confidence |
| @section validation | intent 必须在 16 个枚举值内 |
| @section off-task-detection | 不在 16 步内→引导回 N1 |

## 五、关键纪律

1. 16 份宪法全部使用 field_based 模板——14 必填标签 + changeLevel/changeLevelReason
2. @section topic-boundary 全面省略——偏离检测由 off-task-detection 管
3. changeLevel/changeLevelReason 必填——用于修改日志记录（非 topicEvolution）
4. @section validation 结构化 JSON 是 N5 第3层 DET 复验的规则来源
5. @section output-schema 的 turnType 六值必须与 N5 第2层一致
6. analyzing-session.json 是 ANALYZING 节点的宪法——不是 IN_SESSION 子类型

## 六、常见踩坑

| 坑 | 表现 | 后果 |
|----|------|------|
| 照搬 topic_based 模板 | 写了 @section topic-boundary | 冗余标签，N5 第2层无法解析 |
| 漏写 analyzing-session.json | 只写了 16 份 IN_SESSION 宪法 | N5 第1层找不到宪法文件 |
| changeLevel 写为可选 | field_based 省略 changeLevel | 修改日志无变更级别——规则提炼失效 |
| changeLevelReason 兜底不写"触发 CLARIFYING" | 只写"记录警告" | 与 N5/N6 不一致 |
