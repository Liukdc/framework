## 一、位置与定位

```
N6 数据传递协议 + 守卫规则 + 领域规则存储（已定稿）
  ↓
N7 根宪法  ← 你在这里
  ↓
N8 公共规则
  ↓
N9 各环节宪法
  ↓
N10 tunable 槽位声明
```

**定位**：元智能体的最高法律——不管哪个环节的服务员、哪个接待台的保安，都必须遵守的根本大法。全部 field_based，无松弛节点，无 topicEvolution。

## 四、产出物

N7 产出一样东西：根宪法文件（root-constitution.json），八条不可逾越红线。

### 4.1 根宪法八条

| 编号 | 条款 | 内容 | 不可覆盖 |
|------|------|------|---------|
| 1 | 安全红线 | 禁止生成恶意代码、暴力、色情、政治敏感内容。具体敏感内容范围由 N1 边界清单定义 | ❌ 绝对禁止 |
| 2 | 元指令集 | 定义节点切换指令的子串匹配模式。词表内容：唤醒词"小智"，退出词"小智拜拜""小智退出"，取消词"小智取消这个节点"。N1 元指令词表为权威来源，N7 与 N1 保持同步。N5 第0层直接引用，节点切换路由由 N4 转移图和 N5 调度逻辑定义 | ❌ 不接受覆盖 |
| 3 | 收敛义务 | 所有 LLM 节点必须在当前环节内收敛，输出结构化 JSON | ❌ 不接受覆盖 |
| 4 | 输出格式 | 模型实例返回调度器的结果必须是符合 turnType schema 的结构化 JSON（含 askingField + changeLevel + changeLevelReason）。changeLevel/changeLevelReason 用于修改日志记录——追踪每个环节产出物的变更幅度（major=核心设计变化/minor=渐进调整/invalid=无效变更），供 N10 异步规则提炼消费 | ❌ 不接受覆盖 |
| 5 | 降级链 | L1 结构校验 + DET 值域复验（field_based 值域校验，规则来源 N9 @section validation）+ confidence 检查（冷启动软拦截/非冷启动硬拦截）+ cross_task 跨任务延伸检测（字段白名单）+ 硬编码兜底，五项均不可缩减。冷启动窗口参数由 N10 tunable 清单定义 | ⚠️ 可扩展不可缩减 |
| 6 | 仲裁权 | 调度器 DET 校验结果优先于模型内部校验结果。两者冲突时以调度器 DET 为准 | ❌ 绝对权力 |
| 7 | 偏离守卫 | 三类偏离保护：(1) IN_SESSION 期间用户输入偏离当前步骤时，调度器 DET 关键词扫描辅助触发偏离标记，最终判断由模型给出。词表来源 N1 notDoList，N5 第0.5层执行。(2) 放弃指令匹配——"算了""不做了""先这样吧"等放弃指令触发 giveup 路径。词表由 N7 定义，N5 第0层引用。(3) 前置依赖守卫——N5 在 ANALYZING→IN_SESSION 路由时校验 N4 前置依赖矩阵（DET 0 LLM），跳过未完成步骤时引导至正确环节 | ❌ 不接受覆盖 |
| 8 | 模糊目标追问模板 | 当用户目标不足以匹配 16 个步骤时，追问目标的类型（哪一步）和主题（想设计什么）两个维度。具体场景的追问示例由各环节宪法的 @section ask-rules 定义 | ❌ 不接受覆盖 |

### 4.2 第5条降级链详解

| 项 | 执行体 | field_based 行为 |
|----|--------|----------------|
| L1 结构校验 | DET | JSON.parse + Schema，失败→重试1次→熔断→硬编码回复 |
| DET 值域复验 | DET | 值域校验——规则来源 N9 @section validation |
| confidence 检查 | DET | 冷启动软拦截 / 非冷启动硬拦截 |
| 跨任务延伸检测 | DET | 字段白名单——16步内字段不触发越界 |
| 硬编码兜底 | DET | 熔断后兜底回复落盘 |

## 五、关键纪律

1. 根宪法由设计者写死，不接受运行时动态修改
2. 根宪法八条最低条款必须全覆盖——可扩展不可缩减
3. 第5条降级链五项检查均不可缩减
4. 第5条 DET 值域复验为 field_based 值域校验
5. 第5条 confidence 检查区分冷启动期和非冷启动期
6. 第4条 changeLevel/changeLevelReason 用于修改日志记录（非 topicEvolution）
7. 第6条仲裁权绝对优先——scheduler_det > model
8. 第7条偏离守卫含三类保护——DET 关键词扫描 + 放弃指令词表 + 前置依赖守卫，均不接受局部宪法覆盖
9. 第2条元指令集词表与 N1 同步——N1 为权威来源
10. 根宪法的修改需要全量回归测试

## 六、常见踩坑

| 坑 | 表现 | 后果 |
|----|------|------|
| 第4条照搬基准版 topic_based 描述 | changeLevel 写成"topic_based 模式专属" | 设计者误以为 field_based 不需要 changeLevel——修改日志缺失 |
| 第7条保留松弛节点守卫 | 照搬基准版的"松弛节点↔收敛节点" | N5 尝试实现不存在的松弛节点路由 |
| 第3条写了"松弛节点除外" | 基准版原文未删 | N13 骨架代码生成冗余的松弛节点收敛豁免逻辑 |
| 元指令集词表直接写在 N7 | 在 N7 里定义"小智""小智拜拜" | N1 改了词表但 N7 没同步——N5 第0层引错来源 |

| # | 检查项 | 通过标准 |
