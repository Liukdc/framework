# N8 环节宪法：公共规则编写（元智能体 v2.0·草案）

<!-- @section constitution -->
field_based
taskType 判定理由：公共规则三条为确定性约束——纯JSON输出、坚持真实判断、一次聚焦，不存在语义模糊空间，故归为 field_based。

收敛方向：基于 N7 根宪法 + N5 调度器核心逻辑 + N1 边界清单，编写元智能体所有 LLM 调用必须遵守的三条公共规则。确保：
- 规则1输出纯JSON——对应根宪法第4条和第5条
- 规则2坚持真实判断——对应根宪法第6条精神
- 规则3一次聚焦——对应根宪法第5条和第3条，field_based字段白名单，needSemanticExtensionCheck=false
- 无松弛节点例外声明
- 无 topic_based 分支

- 读取根宪法第4条和第5条——确认规则1的根宪法依据
- 读取根宪法第5条和第3条——确认规则3的根宪法依据
- 读取根宪法第6条——确认规则2的精神来源

**2. 消费 N5 交付物**：
- 读取 N5 第3层 L1 结构校验——确认规则1的执行方
- 读取 N5 第3层跨任务延伸检测——确认规则3的 DET 执行方

**3. 消费 N1 交付物**：
- 全部 field_based——确认规则3用字段白名单，needSemanticExtensionCheck=false

**4. 逐条确认三条规则**（详见 @section ask-rules）

<!-- @section convergence -->

元智能体 N8 产出物：公共规则文件（common-rules.json），含三条规则。

<!-- @section ask-rules -->

**1. 上游消费确认**（askingField="upstream_review"）：
- 展示 N7 根宪法第4/5/6条摘要
- 展示 N5 第3层 L1 结构校验和跨任务延伸检测
- 确认全部 field_based，needSemanticExtensionCheck=false

**2. 规则1——输出纯JSON**（askingField="rule1"）：
- 确认内容：所有 LLM 输出必须是纯 JSON
- 确认执行方：N5 第3层 L1 结构校验
- 确认兜底：失败→重试1次→熔断→硬编码回复
- 确认不受冷启动影响

**3. 规则2——坚持真实判断**（askingField="rule2"）：
- 确认内容：不因回避矛盾而放弃真实判断
- 确认执行方：各环节宪法 prompt 约束，DET 不介入
- 确认与根宪法第6条共享精神

**4. 规则3——一次聚焦**（askingField="rule3"）：
- 确认内容：不主动引入跨步骤延伸话题
- 确认 prompt 层：环节宪法 @section off-task-detection
- 确认 DET 层：N5 第3层字段白名单，needSemanticExtensionCheck=false
- 确认 field_based 字段白名单 DET 即可

**用户确认标准**：显式确认 / 隐式确认 / 沉默跳过

<!-- @section completion -->

同时满足以下条件后返回 turnType="complete"：

1. 三条规则全部编写完毕
2. 规则1执行方为 N5 第3层 L1 结构校验，兜底行为完整
3. 规则2执行方为各环节宪法，DET 不介入
4. 规则3 field_based 字段白名单，needSemanticExtensionCheck=false
5. 规则3无松弛节点例外声明
6. 无 topic_based 分支描述
7. 用户已确认
