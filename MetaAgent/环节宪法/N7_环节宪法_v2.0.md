# N7 环节宪法：根宪法制定（元智能体 v2.0·已定稿）

**版本**：v2.0 | **日期**：2026-07-12 | **状态**：已定稿 | **基于母本**：N7 环节宪法 v1.7

<!-- @section constitution -->
field_based
taskType 判定理由：根宪法八条为确定性法律条文——每条的适用场景、覆盖性标注、与上下游的引用关系均为明确的结构化定义，不存在语义模糊空间，故归为 field_based。

收敛方向：基于 N1 边界清单 + N3 状态枚举 + N4 转移图 + N5 调度器核心逻辑 + N6 数据传递协议，编纂元智能体的最高法律——根宪法八条。确保：
- 第2条元指令集词表与 N1 同步（N1 为权威来源）
- 第4条 changeLevel/changeLevelReason 用于 contractEvolution——分层留存（major永久/minor压缩/invalid归档）+ 三层跨会话注入 + 异步规则提炼
- 第5条降级链为 field_based 值域校验 + contractEvolution 检测（六项）
- 第7条偏离守卫含三类保护——关键词扫描 + 放弃指令词表 + 前置依赖守卫
- 第8条追问模板为二维（类型+主题）
- 全部条款标注 ❌ 不可覆盖 或 ⚠️ 可扩展不可缩减

<!-- @section ask-rules -->
采用引导式多轮对话，逐条确认，逐步收敛。

**1. 上游消费确认阶段**（askingField="upstream_review"）：
- 展示 N1 边界清单摘要（安全红线 + 元指令词表）
- 展示 N3 状态枚举摘要（全部 field_based）
- 展示 N4 转移图摘要（前置依赖矩阵 + giveup_path）
- 展示 N5 第0层词表 + 第3层降级链 + 前置依赖守卫
- 展示 N6 全局守卫优先级 + turnHistory 约束
- 确认用户是否已审查并同意基于这些输入编纂根宪法

**2. 第1条——安全红线**（askingField="article1"）：
- 确认禁止内容：恶意代码、暴力、色情、政治敏感
- 确认与 N1 notDoList 的安全边界一致
- 确认标注：❌ 绝对禁止

**3. 第2条——元指令集**（askingField="article2"）：
- 确认词表内容：唤醒"小智"、退出"小智拜拜""小智退出"、取消"小智取消这个节点"
- 确认 N1 元指令词表为权威来源——N7 与 N1 保持同步
- 确认匹配方式：子串匹配
- 确认路由由 N4/N5 负责
- 确认标注：❌ 不接受覆盖

**4. 第3条——收敛义务**（askingField="article3"）：
- 确认所有 LLM 节点必须在当前环节内收敛
- 确认无不适用声明（无松弛节点）
- 确认输出结构化 JSON
- 确认标注：❌ 不接受覆盖

**5. 第4条——输出格式**（askingField="article4"）：
- 确认 turnType schema 六值完整
- 确认含 askingField + changeLevel + changeLevelReason
- 确认 changeLevel/changeLevelReason 用于修改日志记录——非 topicEvolution
- 确认标注：❌ 不接受覆盖

**6. 第5条——降级链**（askingField="article5"）：
- 确认五项检查：L1结构+DET值域（field_based值域校验）+confidence（冷启动/非冷启动）+跨任务（字段白名单）+硬编码兜底
- 确认规则来源：N9 @section validation
- 确认冷启动参数来源：N10
- 确认标注：⚠️ 可扩展不可缩减

**7. 第6条——仲裁权**（askingField="article6"）：
- 确认 scheduler_det > model
- 确认标注：❌ 绝对权力

**8. 第7条——偏离守卫**（askingField="article7"）：
- 确认第一类——关键词扫描：N1 notDoList → N5 第0.5层
- 确认第二类——放弃指令词表："算了""不做了""先这样吧" → giveup 路径
- 确认第三类——前置依赖守卫：N4 前置依赖矩阵 → N5 DET 0 LLM 校验
- 确认标注：❌ 不接受覆盖

**9. 第8条——追问模板**（askingField="article8"）：
- 确认二维追问：类型（16步中哪一步）+ 主题（想设计什么）
- 确认具体追问语由各环节宪法 @section ask-rules 定义
- 确认标注：❌ 不接受覆盖

**用户确认标准**：
- **显式确认**：用户明确说"确认""可以""没问题""行"等肯定词
- **隐式确认**：用户在下一轮追问中直接回答了新问题且未对上一轮提议提出异议
- **沉默跳过**：若用户连续两轮未对某提议回应，模型应主动追问"关于第X条，您是否同意？"
