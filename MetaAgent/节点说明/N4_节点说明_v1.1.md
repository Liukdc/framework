# N4 节点说明（元智能体 v1.1）
**版本**：v1.1 | **基于母本**：N4 节点说明 v5.6 / MetaAgent L2 v2.0

---

## 一、定位

N4 把 N3 的状态枚举转化为确定性转移图——从每个状态出发，定义什么条件触发跳转到哪个状态。同时建立路由表（intent→环节宪法映射）。

---

## 二、产出物

| 产出 | 说明 |
|------|------|
| 状态转移图 | 所有转移边（from→to + pathType + 触发条件 + topicEvolutionEventAppended） |
| 路由表 | intent→对应环节宪法映射 |

---

## 三、转移边定义

每条转移边含：
- from / to：源状态和目标状态
- pathType：optimal_path（正常）/ off_task_path（偏离）/ giveup_path（放弃）/ exit_path（退出）/ validation_failed_path（校验失败）/ confirm_path（确认）/ chitchat_path（闲聊）/ soft_landing_path（软着陆）/ silent_input（空输入）
- trigger：触发条件（自然语言描述）
- exitType：仅 CLOSING 出边需要——normal（保留_pendingCollectedFields）/ fallback（清空）
- expectsChangeLevel：topic_based 的 topicEvolution 转移边期望模型返回 changeLevel
- topicEvolutionEventAppended：是否在转移时追加 topicEvolution event

---

## 四、元智能体标准转移边

基于 L2 §4.2，最少 15 条转移边。核心边：

**正常流程**：IDLE→[唤醒词]→LISTENING→[用户输入]→ANALYZING→[intent=develop]→IN_SESSION(P0)→complete→DET→WAITING_CONFIRM→confirm→IN_SESSION(N1)→...→IN_SESSION(N15)→complete→EXECUTING→CLOSING→IDLE

**异常流程**：IN_SESSION→off-task→ANALYZING（暂存半成品）/ IN_SESSION→giveup→LISTENING / WAITING_CONFIRM→modify→当前环节 / WAITING_CONFIRM→放弃→LISTENING

**回溯流程**：IN_SESSION→off-task→ANALYZING→{intent:backtrack}→target环节→IN_SESSION

---

## 五、路由表

元智能体固定映射：intent=develop → 环节链 P0→N1→...→N15。用 stepName（非 intent）做第二层模型绑定——因为所有环节都是 develop intent。

---

## 六、关键纪律

1. 所有转移边硬编码，不调 LLM 判断下一状态
2. 每条边标注 pathType + trigger
3. 路由表 intent→constitution 映射完整
4. expectsChangeLevel 从路由表条目读取（非转移边 JSON）
5. 松弛节点路由：如 N3 含 SLACK_NODE，需增加 ANALYZING→SLACK_NODE 转移边
