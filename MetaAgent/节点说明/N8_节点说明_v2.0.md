```

  ↓
```

## 四、产出物

N8 产出一样东西：公共规则文件（common-rules.json），包含 3 条公共规则。

### 4.1 公共规则三条

| 编号 | 规则 | 内容 | 执行方 | 说明 |
|------|------|------|--------|------|
| 规则1 | 输出纯JSON | 所有 LLM 调用输出必须是纯 JSON 格式。根宪法第4条落地版 | N5 第3层 L1 结构校验——JSON.parse+Schema，失败→重试1次→熔断→硬编码回复 | field_based 均输出符合 turnType schema 的 JSON |
| 规则2 | 坚持真实判断 | 当用户认知与分析冲突时，不能为了回避矛盾而放弃真实判断。与根宪法第6条共享精神 | 各环节宪法 @section role 约束模型行为，DET 不介入 | 无论 field_based 还是 topic_based，模型均需坚持真实判断 |
| 规则3 | 一次聚焦，禁止跨任务延伸 | 不得在完成当前步骤后主动引入超出能力范围的延伸话题。根宪法第5条跨任务延伸检测和第3条收敛义务的落地版 | prompt层：环节宪法 @section off-task-detection；DET层：N5第3层字段白名单检查 | field_based 用 DET 字段白名单，needSemanticExtensionCheck=false |

---

## 五、关键纪律

1. 公共规则三条不可增减
2. 规则1由 N5 第3层执行，L1 结构校验不受冷启动影响
3. 规则2由各环节宪法在 prompt 中落地，DET 不判断"什么叫坚持"
4. 规则3 field_based 用 DET 字段白名单，needSemanticExtensionCheck=false
5. 公共规则的修改需要回归测试——影响所有环节宪法

## 七、一句话收 N8

元智能体公共规则三条——规则1输出纯JSON（N5第3层L1校验），规则2坚持真实判断（各宪法prompt约束），规则3一次聚焦（field_based字段白名单，needSemanticExtensionCheck=false）。无松弛节点，无 topic_based 分支。
