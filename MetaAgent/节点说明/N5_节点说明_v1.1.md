# N5 节点说明（元智能体 v1.1）
**版本**：v1.1 | **基于母本**：N5 节点说明 v5.8 / MetaAgent L2 v2.0

---

## 一、定位

N5 定义调度器核心逻辑——当用户输入进入系统后，调度器如何逐层过滤、路由到正确的环节、检验输出。

---

## 二、五层过滤逻辑

### 第0层：全局硬匹配（每轮执行，0次LLM）
- 唤醒词（子串匹配）→ LISTENING
- 退出词 → CLOSING→IDLE
- 取消词 → LISTENING
- 元指令词表由 N7 根宪法定义，此处直接引用

### 第0.5层：DET关键词扫描（每轮执行，0次LLM）
- 检测跨意图词汇
- 输出 offTaskSuspicion 标记（含 keywords + confidence）
- 附加到 ANALYZING 的 contract-in 中供 LLM 参考

### 第1层：意图识别（LISTENING→ANALYZING，1次LLM，快速路由模型）
- 输入：用户原始文本 + offTaskSuspicion
- 输出：{intent, taskType, confidence, extracted}
- 元智能体固定意图：develop（进入环节链）/ backtrack（回溯修改）/ exit / other
- ANTI-FLAP：同一 intent 3次飞转→锁定硬编码→提示"请说清楚意图"

### 第2层：环节内模型自治（IN_SESSION态）
- context-manager.buildPromptContext() 拼接上下文（7个参数）
- 发送给模型实例
- 模型返回 turnType
- 调度器根据 turnType 调度：
  - ask/reply→转发message，继续循环
  - complete→进入第3层DET复验
  - off-task→暂存collectedFields，标记偏离前状态，重新意图识别
  - giveup→LISTENING
  - validation_failed→CLARIFYING（复用模型message转发）

### 第3层：DET值域复验（五项检查，0次LLM）
1. L1结构校验：JSON.parse+Schema校验。失败→重试1次→熔断→硬编码回复
2. DET值域复验：按taskType分支
3. confidence检查：两步DET（数值<threshold+pattern扫词表）。冷启动期软拦截/非冷启动期硬拦截
4. 跨任务延伸检测：按taskType分支
5. 硬编码兜底：不受validation规则影响

### WAITING_CONFIRM守卫
- confirm词表→下一环节IN_SESSION
- modify词表→当前环节IN_SESSION
- 放弃词表→LISTENING
- 其他→ANALYZING（可能是backtrack或off-task）

## 三、context-manager三接口

```javascript
buildPromptContext(contextGraph, turnHistory, currentAskField, currentTaskType, currentTopicId, contractStore, tunableParams) → PromptFragment[]
updateGraph(userInput, modelTurn, turnType, askingField, changeLevel, changeLevelReason, validationResult, taskType) → void
archiveGraph() → void
```

buildPromptContext内部分两种锚点模式：
- field_based：字段级硬门控（DET确定性字符串匹配，保护与askingField同名的历史轮次）
- topic_based：三层注入+embedding两阶段检索（0次LLM）。context-manager内部按currentTopicId从contractStore查询数据

---

## 四、changeLevel处理

- 正常：模型返回→调度器直接使用
- 格式错误（非小写major/minor/invalid）→默认minor+记录警告日志
- 兜底值"minor"和"未提供"应来自常量定义（非硬编码魔法字符串）

## 五、changeLevelReason处理

- 正常：模型返回→调度器提取
- 空值或超100字→DET拦截+默认"未提供"+记录警告日志
- **不标记validation_failed**（不触发CLARIFYING流转）

## 六、冷启动机制

- 四参数冻结+软拦截（confidence<threshold时标记warning不进入CLARIFYING）
- 解冻后PID仅调节confidence_threshold。observation_window为常量。emergency_threshold仅手动调参。pid_kp为比例系数常量
