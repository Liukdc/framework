# N1 节点说明（元智能体 v2.0）

**版本**：v2.0 | **日期**：2026-07-11 | **状态**：已定稿 | **基于母本**：N1 节点说明 v6.1

---

## 一、位置与定位

```
P0（认知加载，不产文件）
  ↓
N1 场景定义 + 边界清单  ← 你在这里
  ↓
[纯 field_based → 跳过 N2，直进 N3]
  ↓
N3 状态枚举+taskType固化
```

N1 画出元智能体的边界——16步设计流程中每一步做什么、不做什么。全部 intent 为 field_based（每步产出均为预定义字段集，DET可验）。

### 4.1 一句话场景定义

"对话式智能体开发工具，通过16步流程引导用户从零完成智能体设计并生成"

### 4.2 边界清单

| 维度 | 做什么 | 不做什么（含处理策略） |

| 交互方式 | 纯文本对话引导 | — |
| 对话范围 | 仅限智能体设计相关讨论 | — |
| 数据处理 | 会话内暂存设计中间态 | — |
| 输出物 | L2设计文档 + L3生成包 | — |

**intent 清单（16个，全部 field_based，全部 topicEvolutionEnabled=false）**：

**松弛节点声明**：默认不启用（slackNodeDeclaration.enabled=false）。

**意图粒度约束**：一个 intent = 一个完整闭环的设计步骤。不要拆碎也不要合并。此约束对 N3 状态枚举有约束力。元智能体全部 field_based，不涉及 topicPath 变浅风险。

**护栏路由实现分工**：N1 声明规则——"不在16节点内→引导回N1，前置未满足→定位已完成环节"；N4 声明转移边的前置依赖条件；N5 执行运行时路由。

**产出物修改与版本管理**：所有 16 步产出物均支持修改迭代。用户可回到已完成的步骤修改产出，修改时记录变更日志（时间戳、变更字段、变更原因、变更级别 major/minor），不覆盖原版本，变更日志作为 N10 异步规则提炼的素材。

### 4.3 护栏策略

元智能体的 16 个环节有严格的前置依赖。护栏按以下规则处理：

**场景类型路由分支**（N1 completion 时执行）：

```
N1 产出边界清单和 intent 清单（每项含 taskType）
  ↓
检查所有 intent 的 taskType 分布
  ↓
├─ 全部 field_based → IN_SESSION(n3)  跳过 N2（字段即边界，无需紧张度测试）
└─ 任一 topic_based → IN_SESSION(n2)  先做边界紧张度测试，再做 N3

N9 生成宪法时同步选择模板：
  - field_based intent → templates/field_based/template.md
  - topic_based intent → templates/topic_based/template.md
      + 加载对应的 topic_based 节点说明模板（N2~N8）
```

**规则一：不在16节点内→引导性硬拒→回归N1**

"小智只负责设计引导，共16个步骤。你想从哪里开始？"

**规则二：在16节点内但前置条件未满足→状态机定位已完成环节→引导至下一个待完成环节**

```
用户输入匹配某环节（如"帮我写N5的调度器逻辑"）
  ├── N5 的前置环节（N1~N4）全部完成？→ 进入N5
  └── 否 → 状态机找到"已完成的最高环节"→ 引导至其下一个
      例：N1+N3完成→引导至N4："N5需要N3和N4先完成。当前N4还没走——我们先设计状态转移图？"
```

**规则三：在16节点内且前置条件已满足→直接进入该环节**

| 模板 | 内容 |
|------|------|
| 引导性硬拒（不在节点内） | "小智只负责设计引导，共16个步骤。（当前在{{stepName}}）" |
| 引导性硬拒（前置未满足） | "{{targetStep}}需要{{prerequisites}}先完成。当前{{nextStep}}还没走——我们先继续？" |

### 4.4 语料收集计划

| 项目 | 内容 |

| 目标数量 | 20-30条"想设计智能体"的用户描述 |
| 来源 | 模型基于场景定义生成——语料在 N3 环节内由用户逐条确认 |

### 4.5 元指令词表

| 类型 | 词表 | 作用 |
|------|------|------|
| 唤醒 | `小智` | 启动元智能体 |
| 放弃当前环节 | `小智取消这个节点` | 退出当前环节，回到环节选择 |
| 退出设计会话 | `小智拜拜` / `小智退出` | 退出整个设计会话 |

### 4.6 tunable 参数适用性

| 类别 | 适用 |
|------|------|
| 通用参数 10 个 | ✅ 全部适用（默认值不变） |
| topic_based 专用 4 个（锚点/粗筛/匹配/压缩） | ❌ 不适用 |
| 异步规则提炼参数 4 个（独立启用，不依赖 topicEvolution） | ✅ 已确认：domain_rule_validation_threshold=0.85 / rule_mining_schedule="0 2 * * *" / rule_mining_batch_threshold=5 / rule_mining_batch_size=16 |

## 五、关键纪律

1. notDo 必须带处理策略，禁止裸写
2. 16个 intent 全部 field_based，全部 topicEvolutionEnabled=false
3. taskType 在 N1 初判、N3 固化
4. 异步规则提炼独立于 topicEvolution，按 stepName 映射，field_based 场景同样适用
5. 每个环节产出物支持修改迭代——修改时记录变更日志（时间戳、变更字段、变更原因、变更级别），原版本保留，变更日志作为规则提炼素材
