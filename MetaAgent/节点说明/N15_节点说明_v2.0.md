## 二、调参信号（12 类）

| # | 信号 | 处置 |
|---|------|------|
| 1 | confidence 拦截率 > 10% | 降低 confidence_threshold |
| 2 | confidence 拦截率 < 1% | 提高 confidence_threshold |
| 3 | 跨任务延伸拦截率异常 | 不调参——回溯 N9 检查 @section off-task-detection |
| 4 | major 占比 < 3% | 回溯 N9 检查 changeLevel 判定规则 |
| 5 | minor 压缩频率过高 | 提高 topic_evolution_minor_compaction_threshold（不适用—元智能体无 topicEvolution） |
| 6 | cold_start 解冻过慢 | 减小 cold_start_observation_window |
| 7 | 上下文拼接命中率低 | 不调参——检查 N5 context-manager 的 DET 匹配逻辑 |
| 8 | 防抖过于敏感 | 提高 antiFlapThreshold |
| 9 | turnHistory 膨胀过快 | 减小 turnHistory_limit |
| 10 | _pendingCollectedFields 堆积 | 缩短 pending_fields_ttl |
| 11 | 规则提炼迟迟不触发 | 不调参——检查骨架代码的 topicEvolution event 追加逻辑（元智能体不适用—检查变更日志记录） |
| 12 | pid 震荡 | 减小 pid_kp |

## 四、调参安全边界

| 规则 | 说明 |
|------|------|
| 单次调参幅度 ≤ 默认值 30% | 超出需人工确认 |
| 冷启动参数组整体冻结 | 观测窗口内不可单独调 |
| 每次调参后回归测试 | 重跑 N14 20 条 case |
| 调参记录完整 | 时间戳/操作者/参数名/旧值/新值/原因 |
