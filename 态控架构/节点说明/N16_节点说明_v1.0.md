# N16 节点说明（v1.0）
**版本**：v1.0
**日期**：2026-07-21
**状态**：已定稿
**对齐文档**：态控架构 v5.6 / 流程指南 v8.3 / 节点说明 v6.1 / N15 节点说明 v1.0
## 一、位置与定位
```
N13 骨架生成
↓
N14 审骨架 + 跑 case + 查机制
↓
N15 调 tunable + 交付
↓
N16 打包交付  ← 你在这里
↓
N17 回溯触发器（运行期迭代）
```
**【人话版定位】**
前面 N12 产出了 L3 JSON（状态机说明书），N13-N15 产出了骨架代码、审计报告和调参快照。N16 的任务是把这些全部打包成一个可以直接分发、安装、使用的包——就像把设计图纸、零件清单和组装说明书塞进一个快递箱，收件人拆开就能用。
**打个比方**：N12 是画图纸，N13 是做零件，N14 是质检，N15 是调试。N16 是装箱发货——把图纸、零件、合格证一起封箱，贴上标签，寄出去。

## 五、工具
```bash
node src-v5.8/n16-package.js \
  --l3 ./agent-output/l3-v5.8 \
  --name my-agent \
  --out ./my-agent-pkg
```

| 参数 | 必填 | 说明 |
|------|------|------|
| `--l3` | 是 | L3 JSON 配置目录路径 |
| `--name` | 否 | 包名（默认从 boundary.json 提取） |
| `--out` | 否 | 输出目录（默认 `n16-output/`） |

## 七、与上下游的关系
| 方向 | 节点 | 关系 |
|------|------|------|
| 上游 | N15 调参交付 | N15 锁定 L3 配置后，N16 接管打包 |
| 下游 | N17 回溯触发器 | 运行期发现问题时，N17 触发回溯到对应设计节点重新修改，修改完成后重新走 N12→N16 管线 |

## 八、约束
- N16 是纯打包操作——不修改任何 L3 配置内容，只做复制和包装
- 打包过程中自动执行 `createAgent` 加载验证，验证失败则拒绝打包
- 宪法文件缺失时不阻断打包（走 SDK 容错模式），但会在日志中警告
- 包名从 `boundary.json` 的第一个 intent 自动生成，也可通过 `--name` 手动指定
