# 小山 (Xiaoshan) — 微观文本编辑器

> 态控架构的第三个参考实现。验证态控"工具分层 + 单房间原子操作"能力。

## 定位

小山是挂载在态控房间里的**必用工具**——在任何文本类房间（写作、代码、笔记、聊天）中，用户说"小山"即可进入微观编辑模式：

```
"小山" → M1 口令 DET 匹配 → 创建独立编辑房间
  ↓
点击定位 → 定位接收模块记录光标位置
  ↓
"删掉后面两句话" → IN_SESSION：LLM 解析语义 + DET 边界检测
  ↓
确认 → EXECUTING 原子写入主文本 → 退出 → 回主房间
```

## 与态控的对应关系

| 小山模块 | 态控组件 | 执行体 |
|---------|---------|:--:|
| 唤醒词 "小山" | M1 口令层 EXACT_MATCH | DET |
| 独立编辑会话 | 房间 + roomConversationLog | 上下文管理器 |
| 点击/光标定位 | contract-in（定位数据） | DET |
| 语义指令解析 | IN_SESSION | LLM + DET 规则模板 |
| 操作历史栈 | sessionCheckpoint stepSnapshots | field_based |
| 原子提交 | EXECUTING 写入 | DET |

## 六模块架构（纯逻辑核心库，不绑定 UI 框架）

| 模块 | 职责 | 状态 |
|------|------|:--:|
| 模式切换 | M1 唤醒词监听 → 创建/退出编辑房间 | 🚧 |
| 定位接收 | 接收点击/光标/语音位置输入 | 🚧 |
| 指令解析 | NLU 意图+槽位填充 + 语义边界检测 | 🚧 |
| 编辑执行 | 对目标文本范围执行 DELETE/INSERT/REPLACE | 🚧 |
| 历史栈 | 结构化操作记录，支持撤销和选择性恢复 | 🚧 |
| 原子提交 | 退出时事务性合并到主文本流 | 🚧 |

## 与 ChatGPT Canvas 的差异

| | Canvas | 小山 |
|---|---|---|
| 进入方式 | UI 按钮/快捷键 | 语音唤醒词，DET 精确匹配 |
| 定位方式 | 鼠标拖选文本范围 | 点击一个点 + 系统推算语义边界 |
| 指令粒度 | "把这段改得更简洁" | "删掉后面两句话" — 自动识别边界 |
| 编辑上下文 | 复用主对话流 | 独立编辑房间，物理隔离 |
| 提交方式 | 实时流式修改 | 原子提交，可撤销回溯 |
