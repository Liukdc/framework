## P0 问题（4 个——必须立即修复）

### P0-1：root-constitution 第5条"六项检查"与母本"五项降级链"冲突

- **文件**：`constitutions/root-constitution.json` line 9（articles[4]）
- **现象**：第5条降级链改为"六项检查：L1结构校验+DET值域复验+confidence检查+跨任务延伸检测+**contractEvolution检测**+硬编码兜底"，`degradationSteps: 6`
- **母本对照**：母本 §7.2 第5条明确"五项降级链（L1结构校验+DET值域复验+confidence检查+跨任务延伸检测+硬编码兜底），均不可缩减"。contractEvolution 是 v4.3 的跨会话记忆机制，不是降级链的一部分
- **问题**：把跨会话记忆机制塞进降级链，违背了母本"五项不可缩减"的铁律。contractEvolution 检测应该在节点 exit 时执行，不是 complete 时的降级链环节
- **修复**：改回"五项检查"，`degradationSteps: 5`，把 contractEvolution 检测移到节点 exit 逻辑中

### P0-2：turnType-schema changeLevel/changeLevelReason 缺 null

- **文件**：`turnType-schema.json` line 8-9
- **现象**：
  ```json
  "changeLevel": {"type": "string", "enum": ["major", "minor", "invalid"]},
  "changeLevelReason": {"type": "string", "maxLength": 100}
  ```
- **母本对照**：母本 §8.1 明确"changeLevel：null | major | minor | invalid，仅在内容发生变更时输出，null 表示无变更"；"changeLevelReason：null | 变更原因说明"
- **问题**：schema 不允许 null，无变更时无法表示——模型被迫每轮都输出一个 changeLevel 值，与母本"仅在变更时输出"的设计冲突
- **修复**：changeLevel enum 加 null，changeLevelReason 加 `"nullable": true`

### P0-3：tunables.json 缺失 v4.6/v4.7 全部新增参数

- **文件**：`tunables.json`
- **现象**：缺失 5 个参数：
  - `strengthens_weight_cap`（v4.6 新增，默认 3，[1,10]）
  - `changelevel_major_sample_rate`（v4.6 新增，默认 0.05，[0.01,0.20]）
  - `critical_room_history_boost`（v4.5 新增，默认 2.0，[1.0,5.0]）
  - `session_checkpoint_ttl`（v4.3 新增，默认 604800，[86400,2592000]）
  - `max_critical_rooms`（v4.7 新增，默认 3，[1,10]）
- **母本对照**：母本 §10 tunable 参数清单明确列出这 5 个参数
- **问题**：v4.5/v4.6/v4.7 的新增参数全部缺失，L3 包的参数清单停留在 v4.4 时代
- **修复**：在 generalParameters 中补充这 5 个参数

### P0-4：state-graph ANALYZING 退出路径同触发双目的地

- **文件**：`state-graph.json` line 40-41（transitions）
- **现象**：`ANALYZING→LISTENING`(exit_path) 与 `ANALYZING→CLOSING`(exit_path) 同一触发条件"用户说退出词"指向两个不同状态，无任何区分字段
- **母本对照**：母本 §5 路由表仅有 `ANALYZING→CLOSING`（intent="exit"）
- **问题**：同一触发条件两个目的地，调度器无法决定走哪个——逻辑矛盾
- **修复**：删除 `ANALYZING→LISTENING` 的 exit_path，或补充区分条件

## 核心发现

L3 包的 JSON 文件整体停留在 **v4.4 时代的参数和机制**，没有同步 v4.5/v4.6/v4.7 的演进。最突出的表现是：

1. **tunables.json 缺失 5 个新参数**——v4.5/v4.6/v4.7 全部新增参数都没补进来
2. **root-constitution 把降级链从五项改为六项**——把 contractEvolution 错误地塞进降级链
3. **turnType-schema 不允许 null**——与母本"null 表示无变更"的设计冲突
4. **state-graph 有逻辑矛盾**——同一触发条件两个目的地

这些问题如果直接用于工程实现，会导致调度器行为与母本设计不一致。建议在 L3 包投入使用前，至少修复全部 P0 和 P1 问题。
