# 态控架构 · §一 架构背景

> **本文件是细则文件**——对应总纲概要的 §一。版本 v5.5。

## 一、架构背景

### 1.1 模型是一个无状态的有自己权重分布概率计算单元

模型的计算能力是有限的，而且受到训练结果约束，权重分布是不均衡的。无论是模型本身的计算资源，还是模型外部的计算载体，还有消耗的能源都是有限的，但是要计算的内容是趋向膨胀的。所有 AI 应用本质是消耗直接能源和间接资源的计算行为，每次单位计算资源的消耗是基本固定的，所以当次计算内容对任务的匹配度越高越好。

AI 应用追求计算性价比的路径：首先找到当次输入任务最匹配的内容，然后去合理分配这些内容，把跟任务最匹配的内容放在最适合的位置，然后按这个标准去分配计算内容。

### 1.2 任何任务皆为受限计算

在实际应用中，不存在"纯无限制的自由计算"。写诗、写小说等创意任务同样具有方向、主题与格式限制。在态控视角下，写诗与修改家具参数都是受限空间内的收敛过程，区别在于收敛空间的连续性——前者是在连续语义空间中采样满足约束的样本，后者是在离散参数空间中定位唯一正确路径。

### 1.3 认知局部性原理（输入侧）

任何信息处理主体（人、LLM、确定性程序），在 t 时刻处理节点 N 时，最优输入集 = N 在当前目标下的最小充分条件集（contract-in（输入契约）），不是全集 Γ。这一条是认知必然：

A 人：工作记忆 7±2 组块，逼人看全局反而信息过载、分析瘫痪。

B LLM（推理阶段）：Lost in the Middle（中间迷失）效应使中部信息召回率显著低于头尾；近因偏置使尾部权重远高于前部；长上下文幻觉率随长度单调递增。喂全局是跟注意力机制对着干。

C 确定性程序（DET）：函数纯度要求 f(x) 只依赖 x，不依赖全局变量。
