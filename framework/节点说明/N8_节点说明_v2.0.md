## 一、位置与定位
```
N7 根宪法（已定稿）
↓
N8 架构机制核查  ← 你在这里
↓
N9 各环节宪法
↓
N10 tunable 槽位声明
```
**【人话版定位】**
N7 定了智能体的"最高法律"（根宪法）和架构机制 M1-M4，N8 的任务是核查这四项架构机制是否在调度器（Scheduler）和确定性执行体（DET，Deterministic Execution Thread）中正确实现。N8 不做模型约束——只检查工程层的实现质量。
**【v5.0 核心变化：原公共规则已取消】**
v5.0 取消了 v4.x 中的"公共规则"（规则1：输出纯 JSON、规则2：坚持真实判断、规则3：一次聚焦一件事）。原三条公共规则的内容已分拆入根宪法和环节宪法：
- 规则1（输出纯 JSON）→ 根宪法第2条（输出格式），由 DET L1 结构校验执行
- 规则2（坚持真实判断）→ 根宪法第3条（收敛义务+真实判断），由环节宪法 @section role 嵌入
- 规则3（一次聚焦）→ 根宪法第1条（环节隔离），由环节宪法 @section off-task-detection 和 M2 共同落地
N8 节点职责从"制定公共规则"改为"核查架构机制 M1-M4 是否在调度器/DET 中正确实现"。
**【与 N7 的关系】**
- 根宪法（N7）定义四条不可逾越红线和 M1-M4 架构机制
- N8 核查 M1-M4 在调度器和 DET 中的工程实现是否与 N7 定义一致
- N8 不产生新的规则——只做核查和验证

### 4.1 架构机制 M1-M4 核查清单
| 编号 | 机制名称 | 约束对象 | N7 定义要点 | 核查项 | 通过标准 |
|------|---------|---------|-----------|--------|---------|
| M1 | 元指令集（Meta-instruction Set） | 调度器第0层 | 口令词表（唤醒/退出/取消/切换四类），EXACT_MATCH 直切 + 语义确认层，不含日常用语 | ① N5 第0层词表是否与 N7 M1 词表完全一致 ② 口令层 EXACT_MATCH 逻辑是否正确实现 ③ 语义确认层（meta_suspect→二次确认）是否正确实现 ④ 是否不含日常用语（如"退出""算了""拜拜"未入口令词表） | 四项全部通过 |
| M3 | 仲裁权（Arbitration Right） | 调度器 | DET 校验结果优先于模型内部校验结果，冲突时以 DET 为准 | ① N5 第3层仲裁规则是否声明 scheduler_det > model ② 冲突处理逻辑是否正确（DET 结果覆盖模型结果） | 两项全部通过 |
| M4 | 节点转换守卫（Node Transition Guard） | DET | 两类守卫：松弛↔收敛转换（置信度≥0.7+放弃指令匹配）+ IN_SESSION 偏离守卫（DET 辅助+模型判断） | ① 松弛→收敛转换守卫（置信度≥0.7）是否正确实现 ② 收敛→松弛转换守卫（放弃指令匹配模式）是否正确实现 ③ IN_SESSION 偏离守卫（DET 关键词扫描+模型判断）是否正确实现 ④ 放弃指令词表是否与 N7 M4 定义一致 ⑤ 关键词扫描词表是否来源 N1 边界清单 | 五项全部通过 |
### 4.2 核查结果等级
| 等级 | 含义 | 后续动作 |
|------|------|---------|
| ✅ 通过 | 该项机制在工程实现中完全匹配 N7 定义 | 无 |
| ⚠️ 部分通过 | 核心逻辑正确但存在非阻断性问题（如词表少一条） | 记录问题，标记为待修复，不阻断下游 |
| ❌ 未通过 | 核心逻辑缺失或与 N7 定义不一致 | 阻断下游，必须回溯 N5/N6 修复后重新核查 |
## 五、关键纪律
1. **N8 不产生新规则**——N8 是核查节点，只验证 M1-M4 的工程实现是否与 N7 定义一致
2. **M1-M4 四项必须全部核查**——不可跳过任何一项
3. **M2 核查必须覆盖 topic_based 无 validation 时的回退行为**——这是常见遗漏点
4. **M4 核查必须确认放弃指令词表不含占位符**——占位符意味着词表未实际定义
5. **核查未通过（❌）阻断下游**——N9 环节宪法编写依赖 M1-M4 的正确实现
6. **核查结果必须有明确的通过/部分通过/未通过标记**

## 七、常见踩坑
| 坑 | 表现 | 后果 |
|----|------|------|
| M1 词表不同步 | N7 定义了新退出词但 N5 第0层未更新 | 用户说新退出词不生效 |
| M2 topic_based 无 validation 回退行为遗漏 | N5 第3层未实现"仅执行 L1 + logprobs 检查" | 无 validation 环节的 DET 复验报错或空转 |
| M2 logprobs 未区分冷启动 | N5 第3层冷启动期也硬拦截 | 初期大量误拦 |
| M4 放弃指令词表仍是占位符 | triggerPatterns 含"（设计者填写：...）" | 收敛→松弛节点转换永远无法触发 |
| M4 DET 关键词扫描词表空 | N6 全局守卫未配置关键词 | IN_SESSION 偏离检测失效 |
| 核查通过但实际未实现 | 核查人员只看代码注释不看实际运行逻辑 | 纸面通过，运行时失效 |
