# 杂碎本 · 极简碎片收集 + 被动定时清理

> **摘要：** 极简碎片收集系统以零摩擦力录入为设计目标，结合被动定时清理机制防止信息堆积，并通过反滥用验证与偏好学习优化清理决策质量。 关键词：fragment collection passive cleanup anti-abuse verification preference learning

## 解决什么问题

日常灵感、临时备忘等碎片信息的管理有四个痛点：记录门槛太高（现有笔记要求先选分类、加标签、建文件夹——念头在操作过程中就丢了）、信息只进不出（碎片堆积成垃圾）、清理行为浪费（用户删掉一条记录被视为终点，系统不从中学习偏好）、"重要"标签滥用（标记太容易导致真正重要的被稀释）。

## 怎么解决的

**核心：极简数据结构 + 防滥用验证 + 被动定时清理 + 从清理行为中学习。**

### 极简数据结构（只有两个字段）

每条碎片只有"名称"和"内容"——没有分类、没有序号、没有标签。支持语音、文本、拍照、截图、文件等多模态输入。用户所想即所记，零认知负担。

### "重要"标记的防滥用验证

用户说"这是重要的"→系统不直接标记，而是触发强制验证步骤，要求完成一个与标记语义无关的操作。验证方式可配置：口述预设密码、完成一道简单算术题、重复一段随机短语等。用户可选择自己习惯的验证方式。验证通过后才标记。确保"重要"标签的稀缺性和真实价值。

### 被动式定时清理（系统主动发起）

在预设时间（可用户自定义，如"每天晚上9点"或"每周日下午3点"），系统自动跳出清理会话，逐条展示碎片摘要，引导用户给出处置指示——已处理的归档、还要用的保留。系统也可根据多模态偏好学习的分析结果，自动建议更合适的清理时间。实现"积极遗忘"，保持碎片库的高价值密度。

### 多模态用户偏好学习

在清理反馈中采集声学特征（语调、语速）+文字内容进行交叉分析。声学特征仅在用户主动触发的清理会话中采集，不用于其他场景。特征数据本地处理，不上传云端。用户可随时在设置中关闭声学特征采集。发现用户语音不耐烦但文字中性→标记为潜在负面样本→自动调整清理频率和时机。系统越用越懂你在什么时候愿意被"打扰"。

## 与体系内其他方案的关系

- 与**小娜**：小娜管理高价值的认知资产（永久锚定类），杂碎本管理低价值的碎片信息（临时记录类）。杂碎本中的碎片在清理时若被用户标记为"重要"，可升级为小娜的临时保护类或永久锚定类
- 与**你好闲**：杂碎本收集碎片灵感（轻量级），你好闲暂存深度推演（重量级）——粒度互补
- 与**记录师**：记录师做兜底全文记录，杂碎本做碎片精炼
