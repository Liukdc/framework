## 一、HumanReadable（白话文详细内容）

### 1. 标准状态集

| 状态 | 类型 | 执行体 | 必选/可选 | 理由 |
|------|------|--------|----------|------|
| IDLE | 稳态 | — | ✅ 必选 | 初始空闲态 |
| LISTENING | 稳态 | 人 | ✅ 必选 | 等待用户输入 |
| ANALYZING | 瞬态 | LLM 轻量 | ✅ 必选 | 意图识别（强制选择+logprobs+S/T/U第二分类维度） |
| IN_SESSION | 稳态容器 | LLM | ✅ 必选 | 环节执行中（5个子类型，可调用DET工具） |
| CLARIFYING | 守护态 | DET | ✅ 必选 | 校验失败追问转发（0次LLM，复用模型message） |
| WAITING_CONFIRM | 守护态 | 人 | ✅ 必选 | delete 是不可逆操作（数据不可恢复） |
| EXECUTING | 瞬态 | DET | ✅ 必选 | 纯写入/删除操作（record INSERT / delete DELETE） |
| CLOSING | 瞬态 | DET | ✅ 必选 | 会话清理 |
| SLACK_NODE | 稳态 | LLM 通用 | ⚪ 可选→启用 | N1 slackNodeDeclaration.enabled=true |

**省略状态**：
| 状态 | 省略理由 |
|------|---------|
| PARAM_PARSING | 合并入 ANALYZING（参数提取在意图识别阶段完成） |
| VALIDATING | 合并入 IN_SESSION + DET 复验（模型第一道 + 调度器第二道） |
| PREVIEWING | 不需要可视化 diff（记账场景无可视化需求） |
| CONSTITUTION_GEN | 不需要动态宪法生成（所有环节宪法静态编写） |
| CONFIRM_CONSTITUTION | 不需要宪法确认（无动态宪法） |

**注**：TEMP_TOPIC 由架构自动支持（v5.1 第二分类维度 T→TEMP_TOPIC），不需要在 N3 特别声明。

### 2. IN_SESSION 子类型

| 子类型 | taskType | topicEvolutionEnabled | executor | triggerCondition | dispatchCriteria |
|--------|----------|----------------------|----------|-----------------|-----------------|
| IN_SESSION(record) | field_based | false | LLM | ANALYZING: intent=record | costLevel=low, canHardcode=false, hasObjectiveAnswer=true |
| IN_SESSION(query) | field_based | false | LLM | ANALYZING: intent=query | costLevel=low, canHardcode=false, hasObjectiveAnswer=true |
| IN_SESSION(delete) | field_based | false | LLM+人确认 | ANALYZING: intent=delete | costLevel=high, canHardcode=false, hasObjectiveAnswer=true |
| IN_SESSION(compare) | field_based | false | LLM | ANALYZING: intent=compare | costLevel=low, canHardcode=false, hasObjectiveAnswer=true |
| IN_SESSION(other) | field_based | false | LLM | ANALYZING: intent=other | costLevel=low, canHardcode=false, hasObjectiveAnswer=false |

### 3. 执行体分派理由

| 状态 | 分派理由（含表达模式举例） |
|------|------------------------|
| ANALYZING | 用户口语表达多样化（"记一笔""帮我查一下""那个删了"），无法用正则穷举所有表达，故不能硬规则化。强制选择+logprobs 保证概率可靠性 |
| IN_SESSION(record) | 用户表达多样化（"猫粮35""记个猫粮35块""昨天午饭25"），无法穷举所有模式，故不能硬规则化。有客观答案（字段集），代价小 |
| IN_SESSION(query) | 查询条件多样（"这个月花了多少""昨天那笔""餐饮多少"），无法穷举。有客观答案（查询结果），代价小。v5.4：可调用DET工具query_records |
| IN_SESSION(delete) | 删除定位条件多样（"删刚才那笔""删昨天的午饭"），无法穷举。有客观答案，但**删除涉及数据丢失，代价大**→需人确认 |
| IN_SESSION(compare) | 对比条件多样（"这个月和上个月""餐饮和交通""昨天猪肉和今天猪肉"），无法穷举。有客观答案，代价小。v5.4：可调用DET工具query_total/query_unit_price |
| IN_SESSION(other) | 架构自动路由 S→SLACK_NODE/T→TEMP_TOPIC/U→追问，不需要场景特有处理 |
| EXECUTING | record 的 INSERT 和 delete 的 DELETE 是确定性操作，可硬规则化，代价小。v5.4修正：仅保留写入/删除，查询/对比在IN_SESSION内通过工具调用完成 |
| CLARIFYING | 复用模型 message 转发用户，不调LLM，确定性转发 |
| WAITING_CONFIRM | delete 不可逆→人确认 |
| SLACK_NODE | 闲聊无客观答案，仅受根宪法约束 |

### 4. 环节内模型实例定位说明

IN_SESSION 内的模型实例属于 LLM 执行体，拥有环节级自治权：
- 调度器不参与逐轮对话
- 只在环节入口（注入宪法+上下文+工具）和出口（收turnType + DET复验）交互
- field_based 模式注入：stepSnapshots + roomConversationLog + contractEvolution规则
- v5.4：IN_SESSION 可调用 DET 工具（function calling）
- v5.5：工具分层发现（必用+清单+选用）

### 5. taskType 固化

所有 intent = field_based，topicEvolutionEnabled = false。

**自动启用**：sessionCheckpoint + contractEvolution + roomConversationLog

### 6. 软着陆检查

N1 softLandingTargets:
- "推荐理财" → targetIntent: "query" → IN_SESSION(query) 存在 ✅
- "闲聊" → targetIntent: "other" → IN_SESSION(other) 存在 ✅
- "非记账问题" → targetIntent: "other" → IN_SESSION(other) 存在 ✅

### 7. 架构机制合规

- rule2_truthfulJudgment: compliant=true（N3 交付物无跨任务建议）
- rule3_oneFocusNoCrossTask: compliant=true（N3 交付物无"建议增加状态""建议修改N1边界"等跨任务建议）

| 执行体分派 | 简单表格 | **dispatchCriteria 结构化字段** |
