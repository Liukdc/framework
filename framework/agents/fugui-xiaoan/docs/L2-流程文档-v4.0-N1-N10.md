## <a id="n1"></a>一、N1 场景定义 + 边界清单 + taskType（任务类型）初判

### 1.1 一句话场景定义（锚点）

"这是一个语音记账工具，用户说出支出，系统记录并支持查询和删除。"

**taskType（任务类型）**：**field_based（基于字段）**——离散参数空间，定位唯一正确值：品类/金额/时间三个字段有客观正确答案，错误可被 DET（确定性程序）值域硬校验验证。

### 1.2 边界清单

| 维度 | 做什么 | 不做什么 |
|------|--------|---------|
| 核心功能 | 记录支出、查询历史、删除记录、比对 | 不自动分类、不推荐理财产品 |
| 交互方式 | 语音唤醒 + 文字输入 | 不支持图片识别、手势操作 |
| 对话范围 | 记账相关的指令 | 不闲聊、不回答非记账问题 |
| 触发方式 | 用户主动唤醒+交代 | 不主动推测、不基于历史行为发起对话 |
| 数据存储 | 本地存储 | 不上传云端、不分析用户数据 |

### 1.3 intentTaskTypeMap（意图-任务类型映射表）

| intent（意图） | taskType（任务类型） | 收敛目标 |
|---------------|---------------------|---------|
| record（记账） | field_based（基于字段） | {category（品类）, amount（金额）, time（时间）} |
| query（查询） | field_based（基于字段） | {timeRange（时间范围）, category（品类）, subType（子类型）} |
| delete（删除） | field_based（基于字段） | {category（品类）, time（时间）} |
| compare（比对） | field_based（基于字段） | {category（品类）, timeRange1（时间范围1）, timeRange2（时间范围2）} |
| exit（退出） | field_based（基于字段） | 无字段采集，直接路由 |
| other（询问） | field_based（基于字段） | 无字段采集，兜底回复 |

### 1.4 护栏策略

| 策略 | 说明 | 示例 |
|------|------|------|
| 硬拒 | 直接告知不支持 | "小安只能记账哦" |
| 软着陆 | 拒绝 + 引导回功能范围 | "小安不会聊天。你需要记一笔吗？" |

### 1.5 松弛节点（slack-node）声明

默认不启用。询问/闲聊用收敛节点硬处理。

### 1.6 元指令口令词表

口令≠自然语言。EXACT_MATCH（精确匹配）直切，0 次 LLM（大语言模型）调用。

| 口令 | 触发 | 
|------|------|
| `小安开账` | 唤醒，进入 LISTENING（监听态） |
| `结束并且退出` | 退出，进入 CLOSING（关闭态）→ IDLE（空闲态） |
| `本次动作取消` | 取消当前操作，→ giveup（放弃）→ LISTENING（监听态） |

自然语言"拜拜""好了""算了"等 → 不走口令匹配。走语义确认层：ANALYZING（分析态）的 LLM（大语言模型）判 meta_suspect（疑似元指令）→ 二次确认后才切换。

### 1.7 @importance（重要性标记）预判

| 层级 | 标记 | 说明 |
|------|:--:|------|
| 房间级 | record-session（记账）= critical（极其重要） | field_based（基于字段）核心收敛环节 |
| 产出物级 | collectedFields（已采集字段）= critical（极其重要） | 记账环节最关键产出 |
| 规则级 | L1（基线层）值域校验规则 | N9 写宪法时确定 |

## <a id="n3"></a>三、N3 状态枚举 + 执行体分派 + taskType（任务类型）固化

### 3.1 状态枚举（8 状态）

| 状态 | 类型 | 含义 | 执行体 | 模型 |
|------|------|------|--------|------|
| IDLE（空闲态） | 稳态 | 空闲，等待唤醒 | — | — |
| LISTENING（监听态） | 稳态 | 待命，等待用户输入 | 人 | — |
| ANALYZING（分析态） | 瞬态 | 意图识别 | LLM（大语言模型） | 通用轻量模型 |
| IN_SESSION（会话中） | 稳态 | 环节执行 | LLM（大语言模型） | 通用模型 |
| CLARIFYING（澄清态） | 守护态 | 校验失败追问转发 | DET（确定性程序） | — |
| WAITING_CONFIRM（等待确认） | 守护态 | 删除确认等待 | 人 | — |
| EXECUTING（执行态） | 瞬态 | 执行操作 | DET（确定性程序） | — |
| CLOSING（关闭态） | 瞬态 | 退出清理 | DET（确定性程序） | — |

### 3.2 执行体分派判据

| 状态 | 执行体 | 为什么 |
|------|--------|--------|
| ANALYZING（分析态） | LLM（大语言模型） | 意图映射有客观答案但无法硬规则化穷举 |
| IN_SESSION（会话中） | LLM（大语言模型） | 字段采集有客观答案但口语解析无法穷举 |
| CLARIFYING（澄清态） | DET（确定性程序） | 纯转发，复用 message（消息） |
| WAITING_CONFIRM（等待确认） | 人 | 删除不可逆，代价大 |
| EXECUTING（执行态） | DET（确定性程序） | 写存储，纯确定性 |
| CLOSING（关闭态） | DET（确定性程序） | 清理资源，纯确定性 |

### 3.3 taskType（任务类型）固化

```
taskTypeFinalized（任务类型最终值）: "field_based（基于字段）"
固化理由：全部 6 个 intent（意图）收敛空间均为离散参数空间。固化后运行时不变。
→ sessionCheckpoint（会话进度检查点）机制启用（ttl（存活时间）= 7 天）
→ contractEvolution（契约演化）按 stepName（环节名）隔离提炼规则
```

## <a id="n5"></a>五、N5 态控调度器核心逻辑 + 上下文管理器（context-manager）

### 5.1 态控调度器完整流程

```
用户输入
    │
    ├─ 口令层：EXACT_MATCH（精确匹配），0 次 LLM（大语言模型）调用
    │   ├─ "小安开账"     → IDLE（空闲态）→ LISTENING（监听态）→ "小安在呢。想记什么？"
    │   ├─ "结束并且退出"  → CLOSING（关闭态）→ IDLE（空闲态）
    │   ├─ "本次动作取消"  → giveup（放弃）→ LISTENING（监听态）
    │   └─ 非匹配         → 放行至 ANALYZING（分析态）
    │
    ├─ 语义确认层：自然语言"拜拜""好了""不记了"
    │   → ANALYZING（分析态）判 meta_suspect（疑似元指令）
    │   → 二次确认："你是想退出吗？退出后进度不保存。"
    │   → 确认→ CLOSING（关闭态）/ 否认→继续
    │
    ├─ DET（确定性程序）关键词扫描（第0.5层，每轮）：
    │   "查询/查一下/多少/总共" → offTaskSuspicion（偏离嫌疑）="query（查询）"
    │   "删除/删掉"             → offTaskSuspicion（偏离嫌疑）="delete（删除）"
    │   标记附加到模型输入，不强制覆盖模型判断
    │
    ├─ ANALYZING（分析态）→ LLM（大语言模型）意图识别
    │   {intent（意图）, subType（子类型）, confidence（置信度）, extracted（已提取字段）}
    │   confidence（置信度）≥80 → 直发 / 60-80 → 反问 / <60 → 引导
    │
    ├─ 任务分发：按 intent（意图）选环节宪法 + 注入 extracted（已提取字段）
    │
    ├─ IN_SESSION（会话中）监督循环：
    │   上下文管理器.buildPromptContext() → LLM（大语言模型）模型实例 → turnType（轮次类型）路由
    │
    └─ DET（确定性程序）值域复验（complete（完成）时）→ 五项降级链
        通过 → EXECUTING（执行态）→ LISTENING（监听态）
        失败 → CLARIFYING（澄清态）硬编码追问 → IN_SESSION（会话中）
```

## <a id="n9"></a>九、N9 环节宪法（6 份）

### 9.1 概览

| 环节 | taskType | importance | 核心产出物 |
|------|:--:|:--:|------|
| intent-recognition | field_based | high | intent |
| record-session | field_based | **critical** | collectedFields |
| query-session | field_based | normal | result |
| delete-session | field_based | normal | result |

### A.2 context-manager 硬门控实现

```
// N13 实现要求：context-manager.buildPromptContext() 内的门控逻辑
// 输入参数：currentAskField（当前追问字段，如 "amount"）
// 门控规则：
//   1. 遍历 turnHistory[]，提取每轮的 askingField
//   2. 若 turnHistory[i].askingField === currentAskField → 该轮被"保护"
//      即使综合匹配度低于截断阈值，也不淘汰
//   3. 被保护的轮次以完整原文形式注入，不以摘要形态
//   4. 门控保护上限：不超过 turnHistory_limit 的 50%

function hardGateProtect(turnHistory, currentAskField, limit) {
  const protected = [];
  const rest = [];
  for (const turn of turnHistory) {
    if (turn.askingField === currentAskField) {
      protected.push(turn);
    } else {
      rest.push(turn);
    }
  }
  const maxProtected = Math.floor(limit * 0.5);
  return {
    hardGateProtected: protected.slice(0, maxProtected),
    candidates: [...protected.slice(maxProtected), ...rest],
  };
}
```

### A.3 _scoreMatch 中文 bigram 分词

```
// N13 实现要求：_scoreMatch 对中文文本做 bigram 分词后计算 Jaccard 相似度
// bigram 分词："猫粮多少钱" → ["猫粮","粮多","多少","少钱"]
// Jaccard = |交集| / |并集|

function _scoreMatch(textA, textB) {
  const bigramA = toBigrams(textA);
  const bigramB = toBigrams(textB);
  const intersection = bigramA.filter(b => bigramB.includes(b));
  const union = [...new Set([...bigramA, ...bigramB])];
  return union.length === 0 ? 0 : intersection.length / union.length;
}

function toBigrams(text) {
  const chars = text.replace(/[^\u4e00-\u9fff]/g, '');
  const bigrams = [];
  for (let i = 0; i < chars.length - 1; i++) {
    bigrams.push(chars[i] + chars[i + 1]);
  }
  return bigrams;
}
// 消费方：context-manager 综合匹配度评分
// 非中文部分（数字/英文/标点）使用精确匹配
```
