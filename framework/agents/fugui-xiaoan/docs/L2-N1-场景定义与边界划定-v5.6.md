# L2-N1 场景定义与边界划定（v5.6）

**项目**：富贵小安
**基线**：态控架构 v5.6 / N1节点说明 v7.0 / N1环节宪法 v8.7
**日期**：2026-07-18
**taskType**：全 field_based

## 一、HumanReadable（白话文详细内容）

### 1. 场景定义

"这是一个语音记账工具，用户说出支出，系统记录并支持查询和删除。"

### 2. 意图清单

| intent | 描述 | taskType | topicEvolutionEnabled | interactionMode | dataHandling |
|--------|------|----------|----------------------|-----------------|-------------|
| record | 记录一笔支出 | field_based | false | voice_input_text_reply | 本地存储 |
| query | 查询消费记录 | field_based | false | voice_input_text_reply | 本地读取 |
| delete | 删除某笔记录 | field_based | false | voice_input_text_reply | 本地删除 |
| compare | 对比支出 | field_based | false | voice_input_text_reply | 本地读取+计算 |
| other | 无法归类的输入 | field_based | false | voice_input_text_reply | 架构自动路由 |

**taskType 说明**：全 field_based——离散参数空间，定位唯一正确值，DET 值域硬校验。
- record/query/delete/compare 是业务 intent
- other 由架构自动路由：S（闲聊）→SLACK_NODE，T（任务导向）→TEMP_TOPIC，U（不确定）→追问

**record 两种模式**（用户显性选择，默认简单模式）：
- 简单模式：category → amount → time（quantity=1, unit=null）
- 细致模式：category → amount → time → quantity → unit（可计算单价 = amount ÷ quantity）

**compare 三种对比类型**：
- time（时段金额对比）：dimension1(时段) + dimension2(时段)
- category（类别金额对比）：dimension1(类别) + dimension2(类别)
- unit_price（单价对比）：item(物品) + dimension1(时段) + dimension2(时段)

### 3. 模型选择

- 第一层（状态级默认）：
  - ANALYZING：DeepSeek（通用轻量模型，强制选择+logprobs）
  - IN_SESSION：DeepSeek（通用模型，可调用 DET 工具）
- 第二层（意图级绑定）：N4 回填（field_based 用默认通用模型，不需要专用模型）

### 4. align 声明

alignScenarioFlag = false（富贵小安不需要意图校准功能）

### 5. 边界清单

| 维度 | 做什么 | 不做什么（含处理策略） |
|------|--------|----------------------|
| 核心功能 | record（field_based）、query（field_based）、delete（field_based）、compare（field_based） | 不自动分类（硬拒）；不推荐理财（软着陆→intent: query）；不记收入（硬拒） |
| 交互方式 | 语音唤醒 + 文字输入 | 不支持图片识别（硬拒）；不支持手势操作（硬拒） |
| 对话范围 | 记账相关指令 | 不闲聊（软着陆→intent: other→架构自动路由 SLACK_NODE）；不回答非记账问题（软着陆→intent: other→架构自动路由 TEMP_TOPIC） |
| 数据处理 | 本地存储用户账目 | 不上传云端分析（硬拒）；不分享第三方（硬拒） |

### 6. 护栏策略

三种护栏策略：
- **硬拒**：直接拒绝，不加引导（如"小安只会记账哦"）
- **引导性硬拒**：拒绝后附带回归当前任务的提示（如"小安只会记账哦。{{anchorText}}"）
- **软着陆**：引导到系统已有能力（如"我可以帮你记，你要先记完这条还是先查？"→intent: query）

护栏策略选择决策树：
```
用户偏离输入，系统有能力且应该去处理吗？
├── 是 → 软着陆（必须绑已规划的 intent）
└── 否 → 当前存在进行中的活跃任务？
    ├── 是 → 引导性硬拒（调度器拼接锚点文本；锚点不可用时降级为普通硬拒）
    └── 否 → 硬拒
```

### 7. 软着陆映射表

| 越界主题 | targetIntent | 说明 |
|---------|-------------|------|
| 推荐理财 | query | 引导到查询已有记录 |
| 闲聊 | other | 架构自动路由 S→SLACK_NODE |
| 非记账问题 | other | 架构自动路由 T→TEMP_TOPIC |

### 8. 护栏模板

- **硬拒模板**："小安只会记账哦。"
- **引导性硬拒模板**："小安只会记账哦。{{anchorText}}"（锚点文本由调度器从 activeSession 拼接，如"当前在记录 [category] 的金额"。activeSession 为 null 时降级为"当前无进行中的任务"）
- **软着陆模板**："我可以帮你记，你要先记完这条还是先查？"

### 9. 语料收集计划

- targetCount：min 20, max 30
- sourcePriority（4条路径）：
  1. 已有同类产品真实用户日志（脱敏后）—— 模式 A
  2. 找 5-8 个非项目组成员做 10 分钟口语模拟 —— 模式 A
  3. 设计者自写（交付前必须用路径 2 替换至少 50%）—— 模式 A/B
  4. 模型生成（推理模型基于场景定义生成）—— 模式 B
- replacementRequirement：没有经过路径 1 或路径 2 语料替换的边界紧张度测试不算完成
- applicableExecModes：["B"]
- execModeDecision：模式 B（纯模型生成）——不能接触外部非项目组成员，无同类产品日志。走 N2 双角色串行+信息隔离

**语料类型分布（20-30条）**：
| 类型 | 数量 | 重点 |
|------|------|------|
| 跨会话续写 | ≥2条 | "继续上次那个记账" |
| 正向指令 | ~10条 | record/query/delete/compare 各覆盖 |
| 明显越界 | ≥2条 | "帮我订机票"→TEMP_TOPIC |
| 边界模糊 | ≥3条 | ANALYZING 准确性关键测试 |
| 口令切换 | ≥2条 | "切断房间"口令 + 忘了口令直接说其他业务 |
| 闲聊(S) | ≥2条 | "今天好累"→SLACK_NODE |
| 有目标无房间(T) | ≥1条 | "帮我分析新能源趋势"→TEMP_TOPIC |
