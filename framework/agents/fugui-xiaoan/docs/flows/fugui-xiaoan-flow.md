# 富贵小安 - 完整业务流程与交互规则汇总

> 产品架构师手工定义，开发智能体按此规格实现。78 段定稿，所有分支均已闭环。

## 一、系统概览与核心原则

富贵小安是一个智能记账助手，其核心流程始于用户通过特定指令（如"小安出来记一下"）唤醒。系统所有后续行为均遵循以下核心原则：

1. **语义分析驱动：** 系统通过语义分析判断用户意图，并计算置信度分数，作为决策依据。
2. **阈值反问机制：** 当置信度不足时，系统会通过反问引导用户明确意图，避免误操作。
3. **必填项追问：** 对于记账、查询、删除等需要明确信息的操作，系统会坚持追问直至获取所有必需字段，或用户主动放弃。
4. **简洁交互：** 系统回复与追问话术力求委婉、简短、清晰。
5. **诚实告知：** 对于无法提供的功能或信息，诚实告知"不知道"或"做不到"，不编造。

## 二、主流程：意图识别与分发

用户唤醒小安并发出指令后，系统进入第一层意图识别。

1. **识别维度：** 系统计算用户指令属于以下四类意图的置信度分数（0-100）：
   - 记账
   - 查询
   - 删除
   - 其他（求助、闲聊、设置等）

2. **决策逻辑：**
   - 最高分 ≥ 80：直接进入对应意图的次级流程执行。
   - 60 ≤ 最高分 < 80：反问确认。例："您是想要查询吗？"
   - 最高分 < 60：反问引导。例："没太明白，您是想要记账、查询还是删除呢？"

3. **界面标识：** 系统顶部始终显示当前记账模式（用户在设置中预设）：
   - 简单模式：显示为"小安 · 简单"（暖金色）
   - 细致模式：显示为"小安 · 细致"（青色）

## 三、记账流程

1. **模式定义：**
   - 简单模式：必填字段为 种类、金额、时间。不采集数量，默认记为1。
   - 细致模式：必填字段为 种类、金额、时间、数量。数量需包含单位。

2. **信息采集与追问逻辑：**
   - 系统解析用户初始语句，提取字段。
   - 按顺序检查并追问缺失的必填字段（顺序：种类 → 金额 → 时间 → 数量(细致模式)）。
   - 追问话术示例（金额缺失）："午饭花了多少钱呢？"
   - 用户回答后，填入对应字段，继续检查下一缺失项。

3. **完成与循环：**
   - 所有必填字段齐全后，记账成功，回复："已记录：[种类] [金额]元"（简单模式）或"已记录：[种类] [数量][单位] [金额]元"（细致模式）。
   - 随后自动询问："已经记好了。还有其他需要记的吗？"
     - 用户继续陈述 → 开启新一轮记账循环。
     - 用户说"没有"/不回应（超时5-10秒） → 结束会话，回复："小安都记好了，拜拜。"
     - 用户说"算了不记了" → 放弃当前记录，回复："好的，那就不记了。拜拜。"

## 四、查询流程

系统进行第二层意图识别，判断查询子类型。

1. **子类型识别：**
   - 单笔查询：查单条明确记录。
   - 汇总查询：查时间段内的总和。
   - 对比查询：比较两个时间段的数据。
   - 模糊查询：根据印象模糊查找。
   - （判断逻辑与主流程相同，采用语义分析+阈值反问）

2. **各子类型处理规则：**

   - **单笔查询：**
     - 必填信息：时间 和 种类（两者必须齐全）。
     - 追问逻辑：缺时间问时间，缺种类问种类。
     - 结果展示："找到了：YYYY年MM月DD日 [种类] [金额]元"。
     - 详情追问：若用户要求"详细点"，有则展示更多字段，无则告知"当时只记了这些"。

   - **汇总查询：**
     - 必填信息：时间范围 和 种类（两者必须齐全）。
     - 追问逻辑：缺时间范围问时间范围，缺种类问种类。
     - 结果展示："[时间范围] [种类] 共[X]笔 合计[Y]元"。

   - **对比查询：**
     - 必填信息：相同的种类，以及两个不同的时间范围。
     - 追问逻辑：缺失任何一项则追问明确。
     - 结果展示：展示双方数据及对比结论（如"[种类]：本月均价比上月贵了Z元"）。

   - **模糊查询：**
     - 规则：直接进行语义匹配，不追问。
     - 结果展示：返回匹配度最高的3-5条记录列表。若无该种类记录，则回复"没找到相关的记录"。

## 五、删除流程

1. 必填信息：时间 和 种类（必须明确到具体记录）。
2. 追问逻辑：缺失任何一项则追问明确。
3. 二次确认：找到匹配记录后，必须展示并确认："找到一笔记录：…… 确认要删除吗？"
   - 用户确认（"是"、"删掉"）→ 执行删除 → 回复"已删除"。
   - 用户否认（"不是"、"算了"）→ 取消删除 → 回复"好的，不删了"。
   - 超时无应答 → 自动取消 → 回复"超时未确认，已取消删除"。
4. 多条匹配：若匹配到多条，列出让用户选择。

## 六、其他流程

处理无法归为上述三类的指令，核心是诚实告知能力边界。

1. **询问功能**（如"你能做什么？"）：回复："小安只能记账哦。您可以说'小安出来记一下'来记账，或者问我查询、删除已有的记录。"
2. **闲聊**（如"讲个笑话"）：回复："小安只会记账，不会聊天哦。您要记账吗？"
3. **涉及设置**（如"换模式"）：引导至设置页面："请到设置页面修改记账模式。这里只能记账、查询和删除哦。"
4. **致谢/结束**（如"谢谢"）：回复："不客气。有其他需要记账的吗？"
5. **无法理解**：回复："没太明白。小安只会记账，您要说'记一下什么'才行哦。"
