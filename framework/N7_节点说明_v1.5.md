**【人话版定位】**
前面 N1-N6 定了"这个智能体能做什么、不能做什么"（边界）、"有哪些房间和走廊"（状态转移）、"每道接待台谁管"（调度逻辑）、"转场带什么清什么"（数据协议）。N7 定的是这栋店面的**最高法律**——不管哪个房间的服务员、哪个接待台的保安，都必须遵守的根本大法。这部法律由设计者（你）亲自写死，不接受任何人在运行时修改。

- 根宪法是系统的**不可逾越红线**，由设计者写死，不接受运行时动态修改

## 四、产出物

N7 产出一样东西：根宪法文件（root-constitution.json），包含以下内容。

### 4.1 根宪法八条（不可逾越的系统红线）

| 编号 | 条款 | 内容 | 可被局部宪法覆盖 | 不可覆盖/缩减的原因 |
|------|------|------|----------------|-------------------|
| 1 | 安全红线 | 禁止生成恶意代码、暴力、色情、政治敏感内容 | ❌ 绝对禁止 | 安全是系统的底线，任何场景不得放松。具体敏感内容范围由 N1 边界清单定义 |
| 2 | 元指令集（口令词表） | 定义节点切换指令的口令词表——含系统前缀或特定组合的专用令牌（如 @@exit、/quit、小安退出），确保在自然对话中极低频。口令层 EXACT_MATCH→直切（0 次 LLM）；自然语言疑似切换→ANALYZING 判 meta_suspect→二次确认后才切（语义确认层） | ❌ 不接受覆盖 | 元指令是系统级口令，局部宪法不得修改。口令词表由 N7 设计者定义，N5 第0层直接引用。口令≠日常用语——"退出""算了""拜拜"等自然语言不入口令词表。节点切换的具体路由规则由 N4/N5 定义 |
| 3 | 收敛义务 | 所有 LLM 节点必须在当前环节内收敛（松弛节点除外），输出结构化 JSON | ❌ 不接受覆盖 | 收敛是态控架构的核心原则——每跳往更确定、更结构化的方向收。如果局部宪法允许 LLM 节点不收敛，整个收敛链将断裂 |
| 4 | 输出格式 | 模型实例返回调度器的结果必须是符合 turnType schema 的结构化 JSON（含 askingField 双模式语义 + changeLevel 预判字段 + changeLevelReason 理由字段，两种 taskType 模式均适用） | ❌ 不接受覆盖 | 统一的 turnType schema 是调度器用同一套逻辑处理所有环节的基础。局部宪法不得修改 turnType 的六值定义 |
| 5 | 降级链 | L1 结构校验 + DET 值域复验 + confidence 检查 + 跨任务延伸检测 + 硬编码兜底，均不可缩减。DET 值域复验按 taskType 分支：field_based 做值域校验，topic_based 做输出格式校验。confidence 检查区分冷启动期（软拦截，标记 warning 不进入 CLARIFYING）和非冷启动期（硬拦截，进入 CLARIFYING） | ⚠️ 可扩展不可缩减 | 五项检查是通用安全底线，任何场景不得缩减。高风险场景可在五项检查后追加 human_takeover 扩展层。冷启动窗口参数（cold_start_observation_window）由 N10 tunable 清单定义，N5 调度器据此执行 |
| 6 | 仲裁权 | 调度器 DET 校验结果优先于模型内部校验结果。两者冲突时以调度器 DET 为准 | ❌ 绝对权力 | 不在安全问题上押注单一概率性判断源。DET 仲裁权是态控"人作为终极容错闭环"的工程落地 |
| 7 | 节点类型转换守卫 | 定义两类守卫：(1) 松弛节点↔收敛节点之间转换的触发条件和守卫规则——松弛节点→收敛节点需置信度≥0.7 方可转换，"放弃指令"匹配模式定义收敛节点→松弛节点的转换触发。具体路由由 N4/N5 实现，但守卫规则由根宪法第7条定义。(2) IN_SESSION 期间用户输入偏离时，调度器 DET 关键词扫描可辅助触发偏离标记，但最终判断由模型给出 | ❌ 不接受覆盖 | 两类守卫均不接受局部宪法覆盖。松弛节点↔收敛节点转换守卫（含置信度≥0.7 和"放弃指令"匹配模式）是根宪法级别的硬约束，局部宪法不得修改；IN_SESSION 偏离守卫的 DET 关键词扫描词表由 N1 边界清单定义，N5 第0.5层执行 |
| 8 | 模糊目标追问模板 | 通用模板：当用户目标不足以分类为可枚举或开放类型时，追问目标的类型、主题、格式三个维度。具体场景的追问示例由各环节宪法的 @section ask-rules 定义 | ❌ 不接受覆盖 | 追问模板的"三个维度"（类型、主题、格式）是通用框架，不可缩减。具体追问语由各环节宪法定制 |

### 4.2 第5条降级链五项检查详解

| 项 | 执行体 | 时机 | 对应公共规则 | 说明 |
|----|--------|------|-------------|------|
| L1 结构校验 | DET | complete 时 | 规则1（输出纯JSON） | JSON.parse + Schema 校验，失败→重试1次→熔断→硬编码回复 |
| DET 值域复验 | DET | complete 时 | — | field_based 做值域校验（金额上限、时间格式、必填字段…），topic_based 做输出格式校验（JSON Schema / 文本格式 / 字数 / 结构…）。规则来源：环节宪法 @section validation 的结构化 JSON。topic_based 无 validation 时仅执行 L1 结构校验 + confidence 检查，跳过此项 |
| confidence 检查 | DET | complete 时 | — | 两步均 DET：①数值 confidence < threshold（默认 0.4）②pattern 扫"硬撑式确认"词表。冷启动期间（cold_start_observation_window 内）采用软拦截——标记 confidence_warning 附加到返回中，不进入 CLARIFYING。冷启动结束后恢复硬拦截。冷启动窗口参数由 N10 tunable 清单定义 |
| 跨任务延伸检测 | DET（默认）/ LLM（例外） | complete 时 | 规则3（一次聚焦，禁止跨任务延伸） | field_based：默认 DET 字段白名单（needSemanticExtensionCheck 通常 false）；topic_based：主题边界检查（可能需要 needSemanticExtensionCheck:true 追加轻量 LLM，可复用 ANALYZING 的轻量模型实例） |
| 硬编码兜底 | DET | 熔断时 | — | 所有校验失败后的最终兜底，硬编码回复。不受 validation 规则存在与否的影响 |

### 4.3 高风险场景扩展

对于医疗预问诊、复杂客服、金融风控等高危场景，在五项检查后可追加 human_takeover 扩展层。五项检查是通用底线，human_takeover 是高风险场景的扩展，不可替换五项检查中的任何一项。

启用条件：仅当场景的"错误代价"判据为"代价大且无客观答案"或"代价大但有客观校验"时，考虑启用 escalated 模式。默认场景仍用五项检查 + 硬编码兜底。


## 五、关键纪律

1. **根宪法由设计者写死，不接受运行时动态修改**——这是"不可逾越红线"的定义
2. **根宪法八条最低条款必须全覆盖**——各场景可扩展但不可缩减
3. **第5条降级链五项检查均不可缩减**——高风险场景只能在五项检查之后追加扩展层，不可替换
4. **第5条 DET 值域复验按 taskType 分支**——field_based 做值域校验，topic_based 做输出格式校验。这是 taskType 双模式在宪法层面的落地
5. **第5条 confidence 检查区分冷启动期和非冷启动期**——冷启动期软拦截，非冷启动期硬拦截。冷启动窗口参数由 N10 tunable 清单定义
6. **第4条输出格式要求 turnType schema 统一**——含 askingField 双模式语义 + changeLevel 预判字段 + changeLevelReason 理由字段（两种 taskType 模式）
7. **第6条仲裁权绝对优先**——调度器 DET 校验结果与模型内部校验结果冲突时，以调度器 DET 为准
8. **第7条定义两类守卫，均不接受局部宪法覆盖**——(1) 松弛节点↔收敛节点转换守卫（含置信度≥0.7 和"放弃指令"匹配模式）(2) IN_SESSION 期间偏离守卫。具体路由由 N4/N5 实现，守卫规则由根宪法定义
9. **第2条元指令集词表由 N7 设计者定义**——N5 第0层直接引用，不得自行修改。节点切换的具体路由规则由 N4/N5 负责
10. **根宪法的修改需要全量回归测试**——影响所有环节宪法和调度器逻辑

## 七、常见踩坑

| 坑 | 表现 | 后果 |
|----|------|------|
| 根宪法条款被局部宪法覆盖 | 某个环节宪法修改了第4条的输出格式 | 调度器无法用统一逻辑处理该环节的 turnType |
| 降级链缩减 | 场景设计者认为"角色扮演不需要跨任务延伸检测" | topic_based 任务的主题边界检查缺失 |
| DET 值域复验未按 taskType 分支 | topic_based 任务套用 field_based 的值域校验 | 诗歌创作结果被检查"金额是否在合理范围"，校验永远失败 |
| 仲裁权被弱化 | 设计者声明"当模型 confidence 高时以模型为准" | 概率性判断源取代确定性程序，安全底线失守 |
| 元指令集词表与 N5 第0层不同步 | N7 加了新退出词，N5 第0层没更新 | 用户说新退出词不生效 |
| 第7条被缩减 | 设计者将松弛节点↔收敛节点转换守卫推出根宪法范围，认为"由 N4/N5 负责" | 置信度≥0.7 守卫和"放弃指令"匹配模式失去根宪法保护 |
| 模糊目标追问模板被局部宪法修改 | 某个环节宪法把"三个维度"缩减为"两个维度" | 追问信息不完整 |
| confidence 检查未区分冷启动/非冷启动 | 冷启动期也硬拦截 | 初期大量误拦 |
| 硬编码兜底被移除 | 场景设计者认为"不需要兜底" | 所有校验失败时系统无响应 |
| 安全红线内容与 N1 边界清单脱节 | N7 定义了安全红线但 N1 没有对应条目 | 安全红线无法落地为具体的护栏策略 |

## 九、一句话收 N7

N7 定下智能体的最高法律——八条不可逾越的红线，由设计者写死。安全红线（第1条）守住底线，元指令集（第2条）定义系统级控制机制（节点切换路由由 N4/N5 负责），收敛义务（第3条）和输出格式（第4条）保证概率性展开的出口被硬卡死，降级链（第5条）按 taskType 分支确保五种错误各有兜底（confidence 区分冷启动/非冷启动），仲裁权（第6条）确保确定性程序拥有最终裁决权，节点类型转换守卫（第7条）定义两类守卫——松弛节点↔收敛节点转换守卫（置信度≥0.7 + 放弃指令匹配模式，两者均不接受覆盖）+ IN_SESSION 偏离守卫（DET 辅助 + 模型判断），模糊目标追问模板（第8条）提供通用信息收集框架。这份宪法是所有环节宪法和调度器逻辑的根基，修改需全量回归测试。
