# 大耳朵 (Big Ears) — 非侵入式智能语音交互输入端

> 态控工具分层体系中的**必用工具**。为所有语音驱动的智能体房间提供统一的语音输入启停协议与自动路由。

## 定位

大耳朵是所有态控智能体的**统一语音入口**。用户说"记事，聆听" → 大耳朵唤醒，转入持续录音 → 录音结束 → 转写文本自动路由到记账房间。它解决的不是"怎么转写语音"，而是"怎么让语音交互的启停过程不打断用户心流"。

与众工具的关系：

```
大耳朵 ← 所有语音驱动的智能体的统一入口
  ↓ 自动路由
proofreader → ① 校对  → ② 进入目标房间（记账/写诗/提醒...）
                        ③ whiteboard → 对照镜 → xiaoshan
                        ④ 屏幕官（阅读）
```

## 核心创新

### 1. 自然语言启停协议——组合唤醒词 + 自动路由

```
用户："记事，聆听"
  → 解析：[功能标识词="记事"] + [唤醒词="聆听"]
  → 启动持续录音
  → 录音结束 → 转写文本自动路由到记账房间

用户："写诗，聆听"
  → 路由到写诗房间

用户："提醒，聆听"
  → 路由到提醒房间
```

### 2. 复合确认智能结束——两套独立逻辑并行

| | 第一逻辑：带倒计时的静默检测 | 第二逻辑：语音结束指令 |
|---|---|---|
| 触发方式 | 静默 5 秒 → 启动 5 秒倒计时 → 超时自动结束 | 用户说"我全部讲完了，请提交" |
| 用户可见 | 进度条/数字倒计时，知道还有几秒 | 无感知 |
| 可中断 | 倒计时期间开口说话即重置 | — |
| 误触发率 | 极低（有倒计时窗口） | 极低（长尾专用短语） |

两套逻辑并行，任意先触发即结束，另一自动失效。

**与现有 VAD 方案的本质区别**：现有方案静默阈值固定，用户无法预知"系统什么时候会掐掉我"。大耳朵将系统内部计时通过倒计时主动公开——用户在 5 秒倒计时内随时可以"再开口"来延续录音，不会被突然掐断。

### 3. 环境质量自检

```
唤醒词识别 → 采集当前环境底噪 → 分析信噪比/清晰度
  → 达标 → 进入持续录音
  → 不达标 → 语音提醒："环境有点吵，靠近一点或换个安静的地方"
```

## 复用性

大耳朵是所有语音驱动的智能体的**共享基础设施**——富贵小安（记账）、提醒房间、写诗房间共用同一套启停协议。新增一个语音智能体，只需在唤醒词映射表加一行：

```
"记事" → { roomId: "accounting", intent: "record" }
"提醒" → { roomId: "reminder", intent: "set_reminder" }
```
