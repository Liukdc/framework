# 校对哨兵 (Proofreader) — 输入预校准与智能纠错工具

> 态控工具分层体系中的**选用工具**。在人机协同输入质量保障中，拦截错误于 LLM 推理之前。

## 定位

校对哨兵是挂载在态控 IN_SESSION 入口处的预处理工具——在用户输入送进 LLM 之前，对语音识别文本做双维度校对。与小山（已生成内容编辑）互补：小山管"出口"，哨兵管"入口"。

## 两种工作模式

### 教导员模式（手动唤醒）

```
"教导员快来" → M1 DET 匹配 → 暂停提交，启动校准窗口
  → 双重校对标注潜在错误
  → 可视化高亮 + 修正候选浮层
  → 倒计时结束/确认 → 提交
```

### 哨兵模式（自动触发）

```
用户输入完成 → 后台静默校对
  → 语音-文字置信度 + 语言逻辑矛盾度 双超阈值
  → 自动弹出短校准窗口（轻量级，不打断）
  → 同上校准流程
```

"教导员请走" → M1 最高中断优先级，立即放弃校准，原文直接提交。

## 核心模块

| 模块 | 职责 | 执行体 |
|------|------|:--:|
| 指令监听 | M1 "教导员快来/请走" 精确匹配 | DET |
| 自动触发监测 | 声学后验概率 + 备选 ASR 对比 + 语法检查 + 上下文矛盾检测 | DET |
| 预校准流程管理 | 倒计时窗口控制，超时强制提交 | DET |
| 智能标注与建议 | 多模态对比定位错误，置信度排序修正候选 | DET + LLM |
| 校准交互界面 | 高亮/浮层标注，点击采纳/语音拒斥 | UI |
| 后继处理 | 校准后文本送 AI 推理 | DET |

## 与态控的对应关系

| 哨兵模块 | 态控组件 | 类型 |
|---------|---------|:--:|
| "教导员快来" | M1 口令层 EXACT_MATCH | DET |
| 暂停提交/倒计时 | IN_SESSION 入口拦截 | DET |
| 语音-文字校对 | ANALYZING 层前置（声学引擎） | DET |
| 语言逻辑校对 | ANALYZING 层前置（轻量 NLU） | DET + LLM |
| 触发决策 | logprobs 阈值裁决对位 | DET |
| 校准窗口 | 临时房间（不可切换） | 上下文管理器 |
