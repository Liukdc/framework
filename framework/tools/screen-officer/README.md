# 屏幕官 (Screen Officer) — 非侵入式沉浸阅读工作台

> 态控工具分层体系中的**选用工具**。在视觉主通道保持专注的同时，通过听觉通道提供辅助解释——"精准解惑"与"保护认知心流"的统一。

## 定位

屏幕官从 AI 对话流中召唤内容，构建一个可自由操控的沉浸式工作台。它的核心创新不是"阅读器多了个语音助手"，而是一套**强制性自律准则**：解释只走语音通道，视觉屏幕强制锁定——保护认知心流不被任何查询响应打断。

与众工具的关系：

```
proofreader → whiteboard → 对照镜 → xiaoshan
（入口）       （中游）      （质量闸门）  （出口编辑）

屏幕官 ← 独立维度：不是编辑工具，是阅读与深度理解工具
          从任意房间召唤内容 → 沉浸阅读 → 非侵入查询 → 回归对话
```

## 五个复合痛点

| # | 痛点 | 现有方案 | 屏幕官的解法 |
|---|------|---------|------------|
| 1 | 长内容回顾难，操控不自由 | 手动翻聊天记录/切换阅读器 | 语音召唤 + 手势自由缩放移动 |
| 2 | 解惑必然导致分心 | 弹窗、链接跳转（视觉入侵） | **纯语音解释 + 屏幕锁定** |
| 3 | 解释来源不可控 | 默认搜索引擎/内嵌词典 | **多源可配：教材/笔记/总结任选** |
| 4 | 验证流程中断心流 | 跳出应用去多个信源交叉验证 | 内置多源检索，不离开界面 |
| 5 | 工具割裂致流程中断 | 屏幕阅读器/语音助手/词典多 App 切换 | 单一工作台解决全部 |

## 核心创新

### 1. 多源可配的纯语音非侵入式解释

```
用户："根据第九版教材，'单克隆抗体'是什么意思？"
  → 解析查询目标："单克隆抗体"
  → 提取来源："第九版教材" → 映射表匹配 → 《医学教材》第九版
  → 从指定知识源检索解释内容
  → 纯语音播报（不弹窗、不高亮、不跳转、不滚动）
  → 屏幕强制锁定 ← 视觉主通道保持绝对静止
```

来源可指向：
- 外部权威词典/教材数据库 → 精确字段匹配
- 个人知识库（阶段总结、断点标签）→ 语义相关性匹配
- 指定人物的结构化观点记录
- 指定书籍/论文

### 2. 强制锁定屏幕——与屏幕阅读器的本质区别

| | 屏幕阅读器 | 屏幕官 |
|---|---|---|
| 目标用户 | 视障用户 | **视力正常的深度阅读者** |
| 视觉通道 | 不需要，语音是唯一通道 | **视觉是主通道，语音是辅助** |
| 屏幕不变的原因 | 用户不需要看 | **主动冻结以保护认知心流** |
| 设计目的 | 替代视觉 | **保护视觉** |

### 3. 语音光标精准定位

```
首次定位："定位到关于[关键词]的段落"
  → 计算逻辑位置 → 映射屏幕坐标 → 朗读前后 10 字确认
  → 用户："不对，再往前一点"
  → 继续微调："往前移动三个字"
  → 直至满意
```

支持从任意字词级别开始的精确锚定——不是"点读"的段落级别。

## 核心模块

| 模块 | 职责 | 关键能力 |
|------|------|---------|
| 内容精准召唤与呈现 | "把阶段总结展示一下" → 全屏无干扰阅读 | 时间/主题/模块/标签四维定位 |
| 多源查询解析与检索 | 解析"根据XX查YY" → 映射知识源 → 检索解释 | 来源映射表 + 语义匹配 |
| 纯语音播报与屏幕锁定 | 解释仅语音通道，屏幕强制冻结 | 核心自律准则 |
| 阅读状态保持与恢复 | 冻结全部阅读状态 → 解释完毫秒恢复 | 渲染缓冲/滚动位置/焦点/Audio 四维 |
| 内容自由操控 | 语音+手势：移动/缩放/翻页/滚动 | 统一互斥操控逻辑 |
| 定向朗读与锚定 | 任意字词级精确起点 + 指定朗读范围 | "往后读一段"、"只读这一段" |
| 辅助功能 | 亮度/色温/防蓝光语音调节 + 标注 + 分屏对比 | 全语音操控 |
