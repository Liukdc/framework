# 态控流程指南 · 四、受控调参系统

> **细则文件**——04/07。版本 v8.3，基于母本 v5.6。

## 四、受控调参系统

### 8.1 系统设置界面

每个宪法中声明的 @section tunable 参数，在智能体的系统设置界面中自动生成配置表单。界面由宪法中的 tunable 声明自动生成，无需手动开发配置页面。每个参数显示其 description、当前值、允许范围/选项。"检测冲突"按钮在保存前执行一次完整冲突检测。

### 8.2 冲突检测引擎

当用户调整参数并保存时，系统遍历该宪法的 conflict_check 规则。规则分两类：`single_param`（range/enum 硬校验）→ 违规硬拒绝；`cross_param`（跨参数语义校验，v4.7 新增）→ 违规默认 warning，弹建议值，允许用户强制保存。cross_param 初始含 5 条规则（上下文预算 / 阈值排序 / 冷启动窗口 / 规则提炼触发 / 置信度冻结），详见总纲 v5.5 §10.1。冲突检测引擎是确定性代码，不调 LLM（single_param 用 json-rules-engine 或 Drools 实现，cross_param 用沙箱 eval 做 compute 表达式求值）。

### 8.3 变更日志规范

每次参数调整必须记录：时间戳、操作者、宪法名称、变更详情（参数名、旧值、新值、调整原因）、冲突检测结果。日志不可篡改，作为审计追溯的最终依据。

### 8.4 调整生效时机

参数保存后，下一轮新会话开始生效。当前正在进行的会话继续使用旧参数值，不受影响。设计理由：避免运行时突然改变规则导致正在进行的会话行为不一致。

### 8.5 与纯 LLM 运行时动态调整的区别

| 维度 | 纯 LLM 动态调整 | 态控 tunable 槽位 |
|------|---------------|------------------|
| 调整方式 | 在 prompt 里写条件 | 在系统设置里改参数 |
| 调整范围 | 无限制 | range/enum 硬限制 |
| 冲突检测 | 无 | 确定性代码检测 |
| 审计 | 难 | 变更日志，每次调整可追溯 |
| 生效时机 | 下一轮对话立即生效 | 保存后下一轮新会话生效 |

**冷启动冻结机制：**

- 冷启动期间参数冻结，但实时观测指标：监测 logprobs 拦截率、topicEvolution event 写入频率、minor 压缩频率、规则提炼触发频率等
- 应急解冻机制：若指标异常（如 logprobs 拦截率 > cold_start_emergency_threshold），触发"应急解冻"——人工确认后调整参数
- 解冻后 P 反馈控制自动调节
- 所有调整记录变更日志
