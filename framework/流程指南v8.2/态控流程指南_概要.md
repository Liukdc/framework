## 二、核心概念速查

### 2.1 taskType 双模式

| field_based | topic_based |
|------------|------------|
| 离散参数空间 → 定位唯一正确值 | 连续语义空间 → 采样满足约束的样本 |
| DET 值域硬校验 | 格式校验 + 语义判断 |
| sessionCheckpoint | topicEvolution |

### 2.2 三类执行体

| DET（确定性） | LLM（概率性） | HUMAN（责任性） |
|----|----|----|
| EXACT_MATCH + 值域复验 + logprobs裁决 + SQL查询 + 文件操作 | 意图识别 + 字段采集 + 偏离判断 + 工具调用决策 | 不可逆决策确认 + 规则确认 |

### 2.3 根宪法 4 条

| # | 条款 |
|:--:|------|
| 1 | 环节隔离——只做本环节任务 |
| 2 | 输出格式——符合 turnType schema 的结构化 JSON |
| 3 | 收敛义务 + 坚持真实判断 |
| 4 | 不编造功能 |

### 2.4 架构机制 M1-M4

| 编号 | 机制 | 核心 |
|:--:|------|------|
| M1 | 元指令集 | 四类口令（wake/switch/exit/cancel）+ S3物理隔离 + roomStateIndex |
| M2 | 降级链 | L1结构+DET复验+logprobs+硬编码兜底 |
| M3 | 仲裁权 | DET 校验优先于模型内部校验 |
| M4 | 节点转换守卫 | 环节间转换条件与守卫 |

### 2.5 编程规则宪法 7 条

| # | 条款 | 核心 |
|:--:|------|------|
| 1 | 执行体分派铁律 | ANALYZING/IN_SESSION=LLM，DET复验/EXECUTING=DET |
| 2 | DET 职责边界 | 确定性计算：EXACT_MATCH + 值域复验 + logprobs裁决 + SQL + 文件操作 |
| 3 | LLM 职责边界 | 意图识别 + 字段采集 + 语义判断 + 工具调用决策 |
| 4 | 降级规则 | 无apiKey拒绝服务，不降级为正则 |
| 5 | 契约对齐 | 严格对齐 N11 契约 schema |
| 6 | tunable/接口隔离 | 阈值从 getTunable() 读，调度器调 context-manager 三接口 |
| 7 | 工具分层发现 | 必用/清单/选用三层 + search_tools + LLM不直接写SQL |

### 2.6 三步法核心

| 步 | 做什么 | 产出 |
|----|------|------|
| 第一步 | 场景理解与边界划定 | 一句话场景定义 + 边界清单 + 边界紧张度测试 |
| 第二步 | 拆状态 | 状态枚举 + 转移图 + 调度器逻辑 + 路由表 + 执行体分派 |
| 第三步 | 写宪法 | 根宪法 + 架构机制 + 环节宪法(18标签)+ 守卫规则 |

## 三、turnType 六值

| turnType | 含义 | askingField |
|----------|------|-------------|
| ask | 模型追问用户 | 字段名/主题标签 |
| reply | 模型回复（非追问） | null |
| complete | 字段齐全/输出满足边界 | null |
| off-task | 检测到偏离 | null |
| giveup | 用户放弃 | null |
| validation_failed | 校验失败 | 失败字段名/主题维度 |
